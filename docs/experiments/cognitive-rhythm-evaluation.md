## 5. 失敗パターン

該当するものを記録する。

- [ ] 問いから入る文章ばかりになった
- [ ] 意外な事実を無理に探した
- [ ] 書き手が持っていない迷い・告白を足した
- [ ] 結論が遅くなった
- [ ] 短い断定文が増えすぎた
- [ ] 節末の予告を消した結果、検索性が落ちた
- [ ] 列挙へ説明を足しすぎ、冗長になった
- [ ] 認知モード分類に時間を使いすぎた
- [ ] 読み進めやすさは上がったが、理解が落ちた
- [ ] 文書ごとの差が減り、同じ型になった

## 6. 判定

### Stable化

次をすべて満たす。

- 6文書中4文書以上で読み進めやすさが改善
- 内容理解と事実保存が現行版を下回らない
- 報告・告知で不適用または部分適用を選べる
- 修正時間が現行比25％以上増えない
- 問い、告白、短い断定が定型化しない

### Experimental継続

- 効果はあるが適用判定が不安定
- 問い・回収台帳の負荷が高い
- 認知モード分類または密度波形の有効性が不明

### 縮小

対象更新テストと問い・回収台帳だけに限定する。

### 撤回

- 理解が改善しない
- 結論や重要条件が遅れる
- 思わせぶりな文章が増える
- 書き手にない経験や感情を作る
- 制作負荷が効果を上回る
