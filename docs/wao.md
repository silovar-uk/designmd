# WAO：期待を意味のある形で上振れさせる

> 状態：stable  
> designmd版：2026.07.19-beta  
> 最終更新：2026-07-19

WAOは、派手さ、奇抜さ、情報量の多さではない。

> 読み手が想定していた体験を、内容と結びついた一手によって、分かりやすさを失わずに上振れさせる。

目立つこと自体を目的にしない。見た瞬間の驚きだけで終わらず、その後の理解、感情、行動、記憶のいずれかを更新する。

## 1. WAOの位置づけ

WAOは、特定の書体、色、動き、レイアウトではない。完成後に加える装飾でもない。

- 問い：何を変えたいか
- 機能：何ができなければならないか
- 編集：どう理解させるか
- WAO：期待をどこで、どう上振れさせるか

機能や正確さを満たした上で、案件固有の驚き、発見、没入、余韻をつくる。

```text
正しいだけ → 使える
整っている → 読める
WAOがある → 思わず見続ける、触る、話す、覚える
```

## 2. 三つのWAO

### 1. 手段のWAO

「そんな場所、媒体、方法でやるのか」という驚き。

- 既存の場所を別の役割へ変える
- デジタルと現場をつなぐ
- 操作、配布、参加、導線そのものを企画にする
- 一度きりの接点を、前後の体験へつなぐ

例：

- 映像の最後に実際の現場音だけを残す
- 二つの場所で対になる表現を置く
- 普段は通過する場所を、物語の入口へ変える

手段の新しさが、目的や読み手と無関係なら採用しない。

### 2. 表現のWAO

「その見せ方は初めて見た」「思わず撮りたくなる」という驚き。

- 見慣れた対象の尺度を変える
- 静かな画面に、一つだけ強い違和感を置く
- 言葉、写真、音、動きの関係をずらす
- 一度目と二度目で意味が変わる表現をつくる

WAOは常に大声である必要はない。

```text
静かな違和感 → 気づく → 意味がつながる → じわっと残る
```

この流れもWAOである。

### 3. 構造のWAO

「別々だと思っていたものが、実はつながっていた」という驚き。

- 前半の情報が後半で別の意味を持つ
- 複数の媒体、記事、ページが一つの物語になる
- 読者の操作によって関係が見える
- 最後に冒頭の見え方が更新される

構造のWAOは、見た目が派手でなくても強い。designmdでは最も再利用価値が高いWAOとして扱う。

## 3. WAOが成立する五条件

### 1. 基準となる期待がある

何を上振れさせるのか説明できる。

- 通常ならどうなるか
- 読み手は何を予想するか
- 今回は何を一段越えるか

基準がなければ、驚きではなくランダムな奇抜さになる。

### 2. 内容と結びついている

別のテーマへそのまま移植できるWAOは弱い。

- 固有の発言
- 場所
- 時間
- 写真
- 数字
- 操作
- 文化

のいずれかと結びつける。

### 3. 分かりやすさを壊さない

理解の前に驚かせる場合でも、重要情報、条件、安全、操作方法を隠さない。

```text
分かる → 引っかかる → 深く分かる
```

を基本とする。緊急告知、注意事項、議事録では、WAOより正確さを優先する。

### 4. 一つに集中している

すべてを大きく、動かし、光らせるとWAOは消える。

- 一画面に一つ
- 一企画に中心となる一手
- 強いWAOの前後に静かな区間

を基本とする。

### 5. その後が変わる

驚いた後に、理解、感情、行動、記憶の何が変わったかを確認する。

- 続きを読みたくなる
- 触って確かめたくなる
- 誰かへ共有したくなる
- 対象の見え方が変わる
- 現場へ行きたくなる
- 後から思い出せる

「一瞬目立った」だけでは完成としない。

## 4. WAOの強度

WAOの強度は三段階で選ぶ。すべてを強くしない。

### 小：気づき

- 細かな言葉の回収
- 触ると関係が見える
- 二度目に意味が増える
- 静かな違和感

長文、業務ツール、説明ページにも使いやすい。

### 中：発見

- 予想外の比較
- ページや媒体をまたぐ接続
- 大きな尺度変更
- 操作による展開

章扉、主要機能、企画の転換に向く。

### 大：体験の転換

- 場所全体を入口へ変える
- 見慣れた対象の意味を反転する
- 複数の接点を一つの物語にする
- 現実の行動を変える

企画全体で原則一つまで。安全、正確性、操作性を犠牲にしない。

## 5. 制作工程への組み込み

### 定める

- 読み手が予想する通常体験を言語化する
- WAOによって何を変えたいか決める
- WAOが不要な領域を明記する

### 集める

- 題材固有の場所、音、発言、数字、行動を集める
- 既存の定番表現を把握する
- 驚きのために創作してはいけない事実を分ける

### 組み立てる

手段、表現、構造から最低一案ずつ出す。

```text
手段のWAO：
表現のWAO：
構造のWAO：
```

三案を足し合わせず、中心となる一案を選ぶ。必要なら別の一案を補助として使う。

### つくる

通常案とWAO案を並べて比較する。

- 何が上振れしたか
- 何が分かりにくくなったか
- 強すぎる箇所はどこか
- WAOの前後に静かな区間があるか

### 確かめる

- 驚きの理由を説明できるか
- 題材固有か
- 一度目と二度目で意味が増えるか
- 内容より演出が前へ出ていないか
- アクセシビリティ、安全、正確さを損ねていないか
- 読後の行動や記憶が変わるか


## 7. 失敗パターン

- 派手な色、巨大文字、アニメーションをWAOと呼ぶ
- 驚きの理由を「目立つから」だけで説明する
- すべてのページへ別の仕掛けを入れる
- 内容と無関係な音、紙、3D、パララックスを足す
- 重要情報を隠してから見せる
- 読者を困らせる操作を意外性と考える
- 既存ブランドや作品の有名な仕掛けを表面的に模倣する
- 一度見たら終わりで、理解や記憶が増えない
- WAOのために事実、発言、感情を創作する
- 強いWAOを連続させ、基準となる静けさを消す

## 8. WAOレビュー

- [ ] 通常ならどうなるか説明できる
- [ ] 何を上振れさせたか説明できる
- [ ] 手段、表現、構造のどこにあるか分かる
- [ ] 題材固有の材料と結びついている
- [ ] WAOを削ると、見栄え以外の何かが失われる
- [ ] 重要情報、操作、安全を邪魔していない
- [ ] 一画面、一企画の中で強さを集中できている
- [ ] 一度目と二度目で意味が増える
- [ ] 読後の理解、感情、行動、記憶のどれかが変わる
- [ ] 「WAOを入れること」自体が目的になっていない
