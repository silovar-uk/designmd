# 図形・矢印・台本レビュー

各項目を0（問題なし）、1（少し該当）、2（明確に該当）で確認します。

## 1. 意味

- [ ] 矢印を「AがBへ何をする」と説明できない
- [ ] 同じ矢印が、時間、因果、受け渡しなど複数の意味を持つ
- [ ] 同じ図形が、主体、課題、成果など別の役割を表す
- [ ] 並列、比較、グルーピングに矢印を使っている
- [ ] 階層や分解へ、意味のない矢印頭を付けている
- [ ] 循環矢印に、実際の戻り先や更新条件がない
- [ ] 分岐条件が書かれていない

## 2. 流れ

- [ ] 一枚の中で、左→右、右→左、上→下が混在する
- [ ] 始点と終点が分からない
- [ ] 矢印が図形へ接続されず、浮いている
- [ ] 線が交差している
- [ ] 一つの線が何度も折れ曲がっている
- [ ] 合流と分岐が区別できない
- [ ] 主な経路が分からない

## 3. 図形

- [ ] 図形の種類が多すぎる
- [ ] ひし形を判断以外に使っている
- [ ] シェブロンを、単なる並列項目に使っている
- [ ] 漏斗を、量が減らない内容へ使っている
- [ ] ピラミッドを、階層や優先以外に使っている
- [ ] 円形配置を、循環しない内容へ使っている
- [ ] 太いブロック矢印を細かな接続へ使っている

## 4. 配置

- [ ] 図形の粒度がそろっていない
- [ ] 図形のサイズが意味なくばらつく
- [ ] 整列、均等配置ができていない
- [ ] 強い関係の要素が遠く離れている
- [ ] 線が文字を横切る
- [ ] 矢印頭が内容より目立つ
- [ ] 色、太さ、大きさへ複数の意味を持たせている

## 5. 台本

- [ ] スライドのノート欄に台本がない
- [ ] 台本が箇条書きの断片だけになっている
- [ ] スライドの文章をそのまま読み上げている
- [ ] 説明文が常体（だ・である調）で統一されていない
- [ ] 引用以外に「です」「ます」「ございます」が残っている
- [ ] 「〜しましょう」「〜してください」など、不要な敬体・呼びかけが残っている
- [ ] 常体と敬体が一つの台本内で混在している
- [ ] 常体にするため、報告書のように不自然に硬くなっている
- [ ] 「である」が機械的に連続している
- [ ] 図のどこから読むか説明していない
- [ ] 主要な矢印の関係を言葉にしていない
- [ ] 色、位置、矢印だけで意味を伝えている
- [ ] 条件、限界、未確認事項を説明していない
- [ ] 前ページからの接続がない
- [ ] 次ページへの橋渡しがない
- [ ] 台本で触れない図形や矢印が残っている
- [ ] 章扉・引用ページを除き、1分程度で話せる説明量がない

文体の詳細は [`speaker-notes.md`](speaker-notes.md) を参照する。

## 6. 戻る段階

- 矢印の意味を説明できない：ストーリーボードへ戻る(STEP 5)
- 図の一文が書けない：構成と論理へ戻る(STEP 5)
- 図形の役割が混ざる：図の文法へ戻る(STEP 8)
- 線が交差する：配置とグリッドへ戻る(STEP 8)
- 台本と図が一致しない：台本または図のどちらかを修正する
- 台本で触れない要素が多い：削減する
- 図を見なければ説明できない：台本を補う
- 台本だけで理解できるが図が役立たない：図を削除または再設計する
- 台本へ敬体が混ざる：引用かを確認し、例外でなければ常体へ統一する
- 常体が硬すぎる：文末は保ちつつ、語彙と文構造を口頭説明へ戻す

## 7. 最終テスト

### 矢印文テスト

すべての矢印を動詞で説明する。

### 削除テスト

矢印を消したとき、失われる意味を説明する。

### 追跡テスト

始点から終点まで、迷わず指でたどる。

### 白黒テスト

色を消しても、形、位置、線で関係を読めるか確認する。

### 音読テスト

台本を読みながら、図を自然な順序で追えるか確認する。

### 文体テスト

引用部分を除き、説明文の文末へ「です」「ます」「ございます」が残っていないか確認する。常体に統一されていても不自然に硬い場合は、話し言葉として書き直す。

### 対応テスト

台本の主要な一文が、図のどの部分に対応するか指せるか確認する。

総合的な品質水準の判定は `docs/quality-rubric.md` のルーブリック(特に⑨レイアウトの意図・⑩写真・図版・図表の必然性)を併用する。
