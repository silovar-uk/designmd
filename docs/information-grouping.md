# 認知単位と副題の設計

> 状態：stable  
> 最終更新：2026-07-16

AIは、内容を漏れなく整理しようとして、見出し、副題、分類、段階、状態名を増やしやすい。情報は精密に見えるが、読み手には「全部にタグが付いた事務資料」として映る。

> 同じ階層で一度に認識させる単位は、原則5つ以下にする。

これは、情報を5件以下へ削る規則ではない。最初に見せる入口を5つ以下へ束ね、詳細は必要な場所で展開する規則である。

## 1. 一度に見せる単位

- 分類、段階、選択肢、カード、タブは原則5つ以下
- 6つ以上ある場合は、3〜5群へまとめてから詳細を見せる
- 同じ階層へ、性質の違う項目を並べない
- 詳細チェックは折りたたみ、付録、制作ログへ送る
- 数をそろえるために、内容を分割・水増ししない

「5」は目標値ではなく、過剰な細分化を止める初期値である。内容上の必然性があれば、3でも4でもよい。

## 2. 副題とラベル

一つの節で使う言語階層は、原則として次の三つまでにする。

1. 見出し
2. 必要な場合だけ補助見出し
3. 本文またはリード

次は削除候補である。

- 見出しを言い換えるだけの副題
- 日本語見出しと同じ意味の英語ラベル
- すべてのカードへ反復する役割タグ
- 内容を増やさないキャッチコピー
- 制作上の分類名を読者へ見せるラベル

英語ラベルは、`PASS 1`、`stable`、固有名称など、実際の識別子として機能する場合だけ残す。

## 3. 5つの上位分類

詳細な既存分類は、次の上位群へ束ねる。

### 制作工程

1. **定める**：読み手、使用場面、読後の変化
2. **集める**：原材料、事実、証拠、未確認事項
3. **組み立てる**：中心メッセージ、構成、役割、情報量
4. **つくる**：視覚方針、代表ページ、実装
5. **確かめる**：AIらしさ、アクセシビリティ、完成検証

従来の詳細なSTEPは、この5段階の内部チェックとして残す。

### ページ・節の役割

1. **見る**：観察、事実、証拠
2. **比べる**：問い、比較、反証
3. **考える**：理論、解釈
4. **決める**：判断、提案
5. **進める**：次アクション、間、転換、回収

詳細タグは制作ログで使い、読者向けの見出しにはしない。

### 学習資料

1. **知る**：定義、識別
2. **分かる**：理由、仕組み
3. **やる**：手順
4. **変える**：状況変化、失敗条件
5. **使いこなす**：転用、自力判断

七つの学習機能は失わず、入口だけを五つへ束ねる。

## 4. 6つ以上を認める条件

次のいずれかを説明できる場合は、6つ以上でもよい。

- 法令、安全、品質検査のため、漏れなく確認する
- 一覧から検索・参照することが目的である
- 順序そのものが対象の仕組みである
- 統合すると異なる責任・条件・意味が混ざる
- 実データの件数であり、分類を人工的に作ったものではない

ただし、読者へ最初に見せる場合は、概要、代表例、検索手段のいずれかを先に置く。

## 5. 検証

### 数え上げテスト

同じ階層に6つ以上ある場合、その必要性を一文で説明する。

### 副題削除テスト

副題を削って意味が変わらなければ削除する。

### 5秒テスト

節を5秒見て、何を伝える節か一文で言えるか確認する。

### 記憶テスト

読後に主要分類を3〜5個で再現できるか確認する。

### 詳細保持テスト

上位分類を減らした結果、必要な手順、例外、責任、失敗条件が消えていないか確認する。
