# アクセシビリティを全工程へ通す

> 状態：stable  
> designmd版：2026.07.15-beta  
> 最終更新：2026-07-15

アクセシビリティは、完成後にコントラストだけを確認する作業ではない。読み手、情報構造、文章、図、ファイル形式、運用までを通して扱う品質条件である。

> designmdはアクセシビリティの検討を支援するが、WCAG、JIS X 8341-3、PDF/UA等への適合を保証・認証しない。

成果物ごとの検証と、必要に応じた専門家レビューは制作側の責任で行う。

## 1. 全媒体に共通する最低条件

- 情報の順序を、位置や色だけに依存させない
- 見出しと本文の階層を意味に合わせる
- 色だけで正誤、状態、分類を区別しない
- 小さすぎる文字や過度に長い行を避け、実寸で確認する
- 図、写真、チャートの要点を文章でも説明する
- リンクや操作の目的を、周辺文脈だけに依存させない
- 読めない、聞けない、操作できない場合の代替経路を用意する
- 専門語、略語、複雑な文を、読み手に必要な範囲で説明する

## 2. STEPごとの確認

| STEP | 確認すること | 成果物 |
|---|---|---|
| 1 目的と読み手 | 閲覧環境、支援技術、投影・印刷・スマホ、字幕等の必要性 | アクセシビリティ前提 |
| 2 原材料 | 画像・音声・動画に代替説明を作れる情報があるか | 代替説明の材料 |
| 3 中心メッセージ | 表現を簡略化しても失ってはいけない意味は何か | 意味の最低線 |
| 4 ストーリー | 見出しだけ、読み上げ順だけでも論理が進むか | 意味順序 |
| 5 ページ設計 | 色・位置・アニメーションなしでも役割が分かるか | 構造設計 |
| 6 情報量 | 認知負荷、専門語、長文、表の複雑さを抑えられるか | 密度予算 |
| 7 ビジュアル | コントラスト、文字、余白、図の代替、色以外の手掛かり | 媒体別基準 |
| 8 実装 | 読み順、代替テキスト、ノート、字幕、タグ、キーボード操作 | 実装済みファイル |
| 9 AIらしさ除去 | 人間らしさの演出が可読性を壊していないか | 修正一覧 |
| 10 検証 | 実寸、白黒、読み上げ、キーボード、別端末で確認 | 検証記録 |

## 3. PowerPoint・スライド

- 投影時に読める文字サイズを実機または想定解像度で確認する
- 色だけで系列や状態を区別しない
- オブジェクトの読み順を確認する
- 意味のある画像・図へ代替テキストを付ける
- 装飾画像は読み上げ対象から外す
- ノート欄で図の読み順、主要な関係、条件、限界を説明する
- 動画には字幕、必要に応じて音声解説を用意する
- アニメーションがなくても内容が成立するようにする

## 4. PDF・文書

- 見出し構造を視覚サイズだけでなく文書構造として持たせる
- 本文、注、表、図の読み順を確認する
- スキャン画像だけのPDFを避け、テキストを選択可能にする
- 表は見出しセルとデータセルの対応が分かるようにする
- リンク文言は「こちら」だけにしない
- PDFとして配布する場合、可能な範囲でタグと代替テキストを確認する

## 5. Web・アプリ

- キーボードだけで主要操作を完了できる
- フォーカス位置が視覚的に分かる
- 見出しレベルを見た目だけで飛ばさない
- ボタンとリンクを役割に合わせて使う
- エラーを色だけで示さず、原因と修正方法を文章で伝える
- 拡大、文字サイズ変更、狭い画面でも内容が失われない
- 動きは停止・回避でき、重要情報を動きだけで提示しない

## 6. 逸脱との優先順位

グリッド、写真、文字の切断、反転などの逸脱は、最低品質を下回らない範囲で使う。

- 読めなくする逸脱は採用しない
- 色だけに意味を預ける逸脱は採用しない
- 読み順を壊す逸脱は、代替順序を用意できない限り採用しない
- 重要情報を隠す演出は採用しない

アクセシビリティは「一つの逸脱ルール」で破ってよい規則ではない。

## 7. 検証の限界

自動チェックだけでは、文章の分かりやすさ、代替テキストの妥当性、読み順の自然さ、字幕の十分さは判定できない。機械検査、人による確認、当事者テストを目的と予算に応じて組み合わせる。
