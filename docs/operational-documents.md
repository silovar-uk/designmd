# 実務文書の差分レビュー事例

> 種別：case note  
> 正典：[`manual.md`](manual.md)  
> 最終更新：2026-07-23

当日運営マニュアルをAI初稿から人が修正した際の差分を記録する。対象は進行表、役割分担表、受付手順、MC台本、チェックリストを含むWord文書。

この文書は新しい正典ではない。共通原則は[`../DESIGN.md`](../DESIGN.md)、Wordマニュアルの規則は[`manual.md`](manual.md)に従う。ここでは、実際の修正から見えた具体例と、既存原則の適用方法を残す。

実務文書の読み手は、制作意図を鑑賞するのではなく、限られた時間で「誰が、いつ、どこで、何をするか」を確認する。編集では情報を立派に見せるより、行動、条件、例外、責任へ最短で到達できることを優先する。

## 人の修正差分から見えた五つの癖

### 1. 説明のための説明を足す

AIは、見出しの直下へ「このページで何を確認するか」を毎回足しやすい。見出しと本文だけで役割が分かる場合、その一文は読者の仕事を増やす。

削除候補：

- 見出しを言い換えただけの副題
- 「まず確認する」「先に固定する」など、ページの進行だけを説明する文
- 「誰が／いつ／どこで／何をするか」など、文書全体の当然の目的を繰り返す文

残す条件：その文がないと、判断順、禁止事項、例外、責任境界のいずれかを誤る。

## 2. 英語を階層として重ねる

`MAIN HALL` と「メインホール」のように、同じ意味の英語と日本語を重ねない。英語は次の場合だけ残す。

- 実際の識別子、製品名、役職名、会場表示である
- 複数言語の利用者が読む
- 運用上、英語表記を照合する必要がある

装飾としての英語は、情報階層を一つ増やす。

## 3. 制作過程を本編へ漏らす

「原資料を再編集した」「このガイドを参照した」「この順番で構成した」などは、制作ログの情報である。現場の読み手が行動を変えない限り、本編へ置かない。

情報は次の四層へ分ける。

1. 本編：当日の行動、条件、責任、例外
2. 台本：読み上げ文、キュー、演出
3. 付録・根拠：原資料、会場図、差分、出典
4. 制作ログ：編集方針、採用・不採用、参照ガイド

## 4. 未確認事項を更新し忘れる

未確認事項は慎重さの装飾ではない。人が事実を確認したら、古い注意書きを残さず、確定情報へ置き換える。

- 原資料同士が矛盾する：未確認として残す
- 担当者が確認した：確定情報へ更新する
- 一部だけ確定した：確定部分と未確認部分を分ける
- 安全、法令、権利に関わる：推測で埋めない

## 5. 文章だけを直し、ページを見ない

段落分けは読みやすさを改善する一方、表の行高と改ページを変える。文面修正後は必ずレンダリングし、全ページを確認する。

特に見る箇所：

- 一行・一行分だけ次ページへ送られた表
- 見出しだけ残り、本文や図が次ページへ送られた箇所
- 大きな空白が生じたページ
- 表の見出し行が次ページで再表示されているか
- ページ数の増減が意図したものか
- ページ番号、ヘッダー、セクション方向のずれ

文章の意味単位と、紙面のまとまりを別工程にしない。

## 編集手順

1. **内容差分**：追加、削除、確定、未確認、表記変更を分類する
2. **読者差分**：その情報が現場の行動を変えるか確認する
3. **階層差分**：見出し、副題、英語ラベル、制作タグの重複を取る
4. **事実差分**：人が確定した情報で古い留保を更新する
5. **視覚差分**：DOCXまたはPDFをレンダリングし、全ページを比較する

## レビューの六問

1. 副題を削ると、行動、条件、例外、責任のどれかが失われるか
2. 英語ラベルは実際の識別子か、それとも装飾か
3. 制作意図や参照ガイドが本編へ漏れていないか
4. 人が確定した情報の近くに、古い「要確認」が残っていないか
5. 段落や改行の修正後、全ページをレンダリングしたか
6. 一行だけのページ、見出しだけのページ、大きな空白がないか

## 例外

- 研修教材では、手順の理由や判断基準を本文へ残せる
- 改訂履歴の提出が必要な文書では、版管理情報を本編または巻末へ置く
- バイリンガル運用では、英語と日本語を並記できる。ただし両者の役割を決める
- 緊急、安全、法令、品質検査では、簡潔さより漏れの防止を優先する
