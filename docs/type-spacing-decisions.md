# タイポグラフィ・余白の目的別判断

> 状態：experimental  
> designmd版：2026.07.15-beta  
> 最終更新：2026-07-15  
> 数値は固定トークンではなく、実寸確認を始めるための範囲である

`visuals.md` の数値を、目的・使用条件・失敗信号と結び付けて使うための試行版である。

## 1. 数値を先に選ばない

次の順で決める。

1. 読み手が何をするか
2. 閲覧距離・端末・時間
3. 情報の優先順位
4. 文字サイズ、行長、行間、余白の仮値
5. 実寸テスト

## 2. タイポグラフィ

| 目的 | 仮の範囲 | 確認すること | 失敗信号 |
|---|---|---|---|
| 投影で一文を強く見せる | 見出し60〜96pt | 後方から読めるか、1資料2〜3回以内か | 全ページが叫んで見える |
| 社内報告を正確に読む | 見出し28〜36pt、本文16〜20pt | 50%表示と投影で読めるか | 情報を詰めるため縮小する |
| A4本文を継続して読む | 本文9〜10.5pt、行長35〜42字 | 印刷と画面の両方で追えるか | 行末から次行を見失う |
| 長文の認知負荷を下げる | 行間1.6〜1.8程度 | 段落のまとまりが残るか | 行間だけ広く、段落差がない |
| 大見出しを塊として見せる | 行間1.1〜1.3程度 | 行が接触しないか | 文字がばらばらに見える |

数値より、フォントの字面、太さ、背景、表示環境を優先する。

## 3. 余白

| 目的 | 使い方 | 注意 |
|---|---|---|
| 関連を示す | 関連項目の間を狭くする | 全間隔を同じにしない |
| セクションを分ける | 段落内より段落間を広くする | 罫線で代替しすぎない |
| 見せ場を作る | 要素を減らし、20〜40%程度へ絞る | 空けるために文字を小さくしない |
| 高密度参照 | 余白を抑え、見出し・列・罫線で検索性を作る | 密度を美徳にしない |
| タッチ操作 | 操作対象間に誤操作を防ぐ間隔を置く | 見た目だけのコンパクト化を避ける |

## 4. 近接の検証

- 見出しと本文は、次の節より近いか
- 注釈は対象と近く、本文とは区別できるか
- 同じ余白単位を使いながら、関係の強弱があるか
- 画面幅が変わっても、グループが崩れないか

## 5. アクセシビリティの下限

このページの範囲はアクセシビリティ適合値ではない。文字サイズ、コントラスト、拡大、読み上げ、操作領域は媒体ごとの基準と実装条件で別途確認する。

## 6. stable化の条件

- 3種類以上の媒体で試す
- 実寸テストの記録がある
- 数値が目的より先に使われていない
- `visuals.md` と矛盾しない
- 読み手テストで改善が確認できる

条件を満たさない場合、このページは補助資料のまま維持するか撤回する。
