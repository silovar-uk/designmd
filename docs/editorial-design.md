# 編集デザイン：雑誌のような読み方を設計する

> 状態：stable  
> designmd版：2026.07.19-beta  
> 最終更新：2026-07-19

「雑誌のようなデザイン」は、明朝体、段組み、紙の粒子を足すことではない。

> 編集者の視点で情報に優先順位を与え、読者がどこから入り、どこで立ち止まり、どの順番と速度で理解するかを設計する。

雑誌らしさは、一枚の装飾ではなく、複数のページや画面を通した編集判断にある。この文書では、その判断を五つの編集型と、全型に共通する原則へ分ける。

## 1. 先に決めること

見た目を選ぶ前に、次の三点を決める。

1. 読後に何を変えたいか
2. 最も強い原材料は何か
3. どのような読み方を求めるか

### 読後の変化

- 重要度を把握する
- 世界観を感じる
- 深く考える
- 文化や態度に触れる
- 仕組みを理解する

### 強い原材料

- 事実、発言、ニュース
- 写真、人物、場所
- 長文、議論、思想
- 声、文化、違和感
- 数字、比較、構造

### 読み方

- 拾い読む
- 連続して眺める
- 腰を据えて読む
- 予測できない展開を楽しむ
- 順番に理解する

## 2. 五つの編集型

五つは完成済みテンプレートではない。案件ごとに主型、副型、一度だけの逸脱を決める。

```text
主型：全体の読み方を決める
副型：一部の章やページで補助する
逸脱：内容上の転換を一度だけ強く示す
```

### 1. 報道・新聞型

**目的：何が重要かを素早く理解させる。**

向く内容：ニュース、試合レポート、業界動向、社会・歴史、複数記事の編集。

設計の中心：

- 大見出し、関連記事、短報の優先順位
- 多段グリッドとモジュール
- 日付、場所、筆者、出典などのメタ情報
- 写真、本文、図表、引用の複数の入口
- 密度の高いページと、要点を止める余白

失敗信号：

- すべての記事が同じ大きさ
- カード一覧になり、トップ記事がない
- 罫線と段組みだけで新聞らしさを作る
- 速報性の演出が事実確認より前へ出る

### 2. 写真・ラグジュアリー型

**目的：人物、商品、場所の世界へ入らせる。**

向く内容：人物インタビュー、ブランドストーリー、写真企画、建築、旅行、ファッション。

設計の中心：

- 写真が文章の前提をつくる
- 全面写真、裁ち落とし、連続写真の間
- 大きな余白と少数の文字
- 低密度ページを途中に置く
- 写真部分と本文部分で光沢感や密度を変える

失敗信号：

- 素材写真が弱いまま余白だけを増やす
- 細い書体と金色だけで高級感を作る
- 写真が本文の飾りになっている
- 全ページを静かにして変化がない

### 3. 文芸・思想誌型

**目的：文章を読み、考える時間をつくる。**

向く内容：エッセイ、批評、歴史、ロングインタビュー、議論の記録。

設計の中心：

- 長時間読める本文組み
- 脚注、傍注、引用、章扉
- 落ち着いた余白と抑制された色数
- 事象と心象の往復
- 本文の流れを壊さない図版や挿絵

失敗信号：

- 文章中心を、小さい文字の詰め込みと誤解する
- 文学らしさを明朝体、茶色、古紙へ短絡させる
- すべての節に比喩的な見出しを付ける
- 長文の中に別の入口や休止がない

### 4. カルチャー・インディー型

**目的：媒体や書き手の態度を示す。**

向く内容：音楽、映画、アート、サブカルチャー、ユーモア、主観の強いコラム。

設計の中心：

- 大胆なタイポグラフィ
- コピー、切り貼り、手書きなど制作痕跡
- 予想外の配置とページ転換
- 基本グリッドの上で一部を崩す
- 本文は読める状態を保ち、扉や写真で摩擦をつくる

失敗信号：

- ランダムに傾け、重ね、汚す
- すべてのページで別の奇抜さを使う
- 読みにくさを人間らしさと考える
- 題材固有の文化より、ZINE風の記号が前へ出る

### 5. 解説・データ型

**目的：仕組み、因果、数字を理解させる。**

向く内容：データ分析、制度解説、年表、戦術分析、調査記事、比較記事。

設計の中心：

- 地図、グラフ、年表、注釈付き写真
- 一度に見せる関係を絞る
- 色、線、重なりによる分類
- 数字を文章から切り離さず、根拠の近くへ置く
- スクロールやページ送りで情報を段階的に開示する

失敗信号：

- 方眼紙や設計図風の装飾だけを加える
- 座標、数字、線が意味を持たない
- 動きが理解より目立つ
- 正確な値を印象的な図だけへ置き換える

## 3. 全型に通底する七原則

### 1. 編集者の視点を置く

すべてを同じ大きさで並べず、重要、補足、転換、余韻を区別する。

### 2. 読み始める場所を明確にする

見出し、写真、リード、引用、キャプションなど、本文以外の入口を設計する。

### 3. グリッドと例外を共存させる

例外を成立させるために基本グリッドを持つ。グリッドは同じ箱を反復する規則ではなく、関係を保つ骨格である。

### 4. 情報量に波をつくる

密と疎、速と遅、説明と余韻を切り替える。すべてを強くしない。

### 5. 文字を内容の声として使う

書体名より先に、断定、静けさ、記録、親密さなど、必要な声を決める。

### 6. 写真と文章を説明し合わせる

写真が文章の前提をつくる、文章が写真の意味を変える、キャプションが新しい事実を足す状態をつくる。

### 7. 共通性と変化を分ける

ロゴ、基本書体、余白、柱などを固定し、特集や記事ごとに色、写真処理、見出し、密度を変える。

## 4. 主型・副型・逸脱の決め方

```text
主型：
主型を選ぶ理由：
副型：
副型を使う章・ページ：
固定するもの：
変えてよいもの：
一度だけの逸脱：
逸脱が示す内容上の転換：
```

例：

```text
主型：報道・新聞型
理由：複数の事実と論点の重要度を拾い読みで伝えるため
副型：解説・データ型
使用箇所：販売推移と来場者属性の章
固定：基本グリッド、本文書体、メタ情報の位置
変化：記事の密度、写真サイズ、引用の扱い
逸脱：人物インタビューの扉だけ全面写真
意味：事実の整理から、当事者の声へ視点を切り替える
```

## 5. 媒体への変換

### PDF・冊子

- 見開きで情報の関係を確認する
- 綴じ、ノド、裁ち落とし、ページの厚みを前提にする
- 前後ページの密度差を確認する

### Web

- 紙面を縮小して再現しない
- 見開きは、横スクロール、重なり、章の切り替えへ翻訳する
- JavaScriptがなくても本文を読める構造にする
- スワイプ以外に前後ボタンと現在位置を用意する
- 動きは注目、関係、変化、転換のいずれかを説明するときだけ使う

### スライド

- 一枚の完成度より、連続するページの速度を設計する
- 静、標準、密、動の代表ページを先に試す
- 発表用では、本文量より口頭説明との役割分担を優先する

### スマートフォン

- 多段組みを一列へ崩すだけにしない
- 入口、本文、補足の順序を再編集する
- 320 CSS px幅でも横方向の内容が切れないか確認する
- 操作対象は十分な大きさと間隔を持たせる

## 6. レビュー

- [ ] どの記事・ページから読むべきか分かる
- [ ] 重要度が文字サイズだけでなく構造にも現れている
- [ ] 密・疎、速・遅の変化がある
- [ ] グリッドが何を関係づけているか説明できる
- [ ] 例外が内容上の転換と結びついている
- [ ] 写真や図を削除した場合に失われる情報を説明できる
- [ ] 主型と副型を、見た目ではなく読者の行動から説明できる
- [ ] 五つの型を固定テンプレートとして使い回していない
