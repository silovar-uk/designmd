# 共通原則

## 人間らしさは雑さではなく判断

手書き風フォント、紙の質感、不規則な余白、口語を機械的に加えない。それらは理由がなければ「人間らしさの装飾」にすぎない。

人間らしさは、例の選択、事実の重みづけ、留保、削除、例外、順序などの判断から生まれる。

## 目指す品質

本ガイドは、次の四つを一つの制作思想として統合する。

- 熱量と記憶に残る企画性
- 抑制と必然性のある余白
- 明晰さと論理の一貫性
- 人間の編集が持つ緩急と不均質

熱量と論理、抑制と記憶性は両立する。

## 完成を一度に求めない

原材料から完成品へ一気に進むと、最初に選ばれた無難な方向が、文章、構成、レイアウト、画像へ連鎖する。

制作は次の五段階へ分ける。

1. 定める
2. 集める
3. 組み立てる
4. つくる
5. 確かめる

詳細な確認項目は各段階の内部へ置き、入口で十数個の工程を並べない。詳しくは [`staged-production.md`](staged-production.md) を参照する。

## 認知単位は原則5つ以下

分類、段階、選択肢、カード、タブなど、同じ階層で一度に認識させる単位は原則5つ以下にする。

情報を削る規則ではない。最初の入口を3〜5群へまとめ、詳細は下位のチェック、付録、制作ログへ送る。

- 6つ以上ある場合は、必要性を一文で説明する
- 副題を削って意味が変わらなければ削除する
- 日本語見出しと同じ意味の英語ラベルを重ねない
- 制作上のタグを読者向け見出しへ流用しない
- 数をそろえるために内容を分割しない

例外と検証方法は [`information-grouping.md`](information-grouping.md) を参照する。

## 原材料と確信度を守る

実際の発言、日時、場所、人数、数値、固有名詞、失敗、迷い、反応、例外、予想外だった点を残す。

次を同じ確信度、同じ見た目で扱わない。

- 事実：確認できたもの
- 解釈：事実から読み取ったもの
- 仮説：まだ検証していないもの
- 提案：今後行うべきこと
- 演出：理解や印象を補助するもの

## 一貫性と均質性を区別する

書体、色、余白、表記、出典形式は一貫させる。文字量、構図、見出しの強さ、項目数、結論の調子は内容に応じて変える。

秩序を作った後で、内容上の転換と結びついた逸脱を置く。逸脱は変化をつけるためではなく、意味の変化を可視化するために使う。

参照資料からは、外見ではなく操作を借りる。

- 骨格：グリッド、組版、階層
- 逸脱：スケール、切断、重なり
- 時間：速度、間、反復
- 固有素材：発言、写真、表、現場記録

## 媒体と読み手に合わせる

- ウェブ：読み手が自分の速度で選択する
- スライド：発表者が時間軸を制御する
- 文書：単独で読んでも理解できる
- SNS：文脈が切り取られることを前提にする

別媒体の見た目だけを移植しない。反証と限界は、最後の免責ではなく主張の近くに置く。

## AIくささの5群

従来の九つの診断要因は、入口では次の五群へまとめる。旧番号は失敗パターンとの対応を保つため、下位分類として残す。

### 1. 言葉と材料が平均化する

含む要因：`1 言語的平均化`、`5 原材料と生活痕跡の欠如`

- 抽象語が多く、具体的な場面が少ない
- 同じ文長、接続詞、結論調が続く
- すべての見出しに副題やキャッチコピーが付く
- 発言、日時、数字、失敗、迷いが消える

対策：固有の材料を戻し、情報の役割に応じて文型・長さ・密度を変える。

### 2. 形式が内容より先に立つ

含む要因：`2 ジャンルと文化の不整合`、`4 装飾と意味の乖離`

- ウェブのカードUIを別媒体へ移植する
- 読み手が理解しない英語ラベルを装飾で使う
- 図解が本文の再掲にとどまる
- 画像、アイコン、囲みを消しても理解が変わらない

対策：媒体、使用場面、読み手、情報の関係から形式を選ぶ。

### 3. 構造が均一・過密になる

含む要因：`3 構造的均質化`

- 同じレイアウトやカードが続く
- 内容を必要以上に細かく分類する
- 同じ階層に6つ以上の段階・副題・選択肢を並べる
- 本文より見出し体系の方が複雑になる

対策：上位を3〜5群へ束ね、詳細は下位へ送る。ページや段落の役割から形式を選ぶ。

### 4. 作り方が前へ出る

含む要因：`7 早すぎる収束`、`8 制作工程の漏出`

- 原材料から、いきなり完成版を作る
- 工程を細分化しすぎ、制作そのものが目的になる
- QUESTION、FACT、CAUSEなどの内部タグが本編へ出る
- 読者が内容より制作会議を先に読まされる

対策：制作は五段階へ束ね、詳細チェックは制作ログへ置く。本編、台本、付録、制作ログを分ける。

### 5. 人の判断と機能が消える

含む要因：`6 判断主体と責任の不在`、`9 再設計による機能喪失`

- なぜその結論、順序、画像なのか説明できない
- 不採用案と最終判断者がいない
- 旧版にあった手順、具体例、失敗条件が消える
- 情報量の減少を改善と誤認する

対策：採用・不採用理由と責任者を残し、再設計前に機能保存表を作る。

具体例は [`ai-failure-patterns.md`](ai-failure-patterns.md) を参照する。

## AIの役割を限定する

AIへ任せやすいのは、候補、抜け、校正、形式、比較軸、反論、体裁である。

人間が保持するのは、目的、読み手、原材料、優先順位、結論、採否、責任である。

AIへ結論、構成、表現、評価を同時に任せない。対話できる場合は重要な段階で止め、ユーザーの判断を得てから次へ進む。
