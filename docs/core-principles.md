# 共通原則

## 人間らしさは雑さではなく判断

手書き風フォント、紙の質感、曲げた線、不規則な余白、口語を機械的に加えない。それらは理由がなければ「人間らしさの装飾」にすぎない。

人間らしさは、例の選択、事実の重みづけ、留保、削除、例外、順序などの判断から生まれる。

## 完成を一度に求めない

原材料から完成品へ一気に進むと、最初に選ばれた無難な方向が、文章、構成、レイアウト、画像へ連鎖しやすい。

完成品を一度に生成するのではなく、次の判断を分けて確定する。

1. 目的と読み手
2. 原材料と証拠
3. 参照先と借りる操作
4. 構成仮説
5. 内容の設計図
6. グリッドとデザインシステム
7. 代表ページ
8. 全体制作
9. 最終レビュー

各段階で、採用理由と不採用理由を残す。構成が未確定の段階で装飾を作り込み、代表ページを検証する前に全体を量産しない。

詳細は [`staged-production.md`](staged-production.md) を参照する。

## 原材料を消しすぎない

実際の発言、会話、日時、場所、人数、数値、固有名詞、失敗、迷い、反応、例外、予想外だった点を残す。

抽象化は必要だが、すべてを「価値」「体験」「本質」「可能性」へ変換しない。

## 事実・解釈・提案を分ける

- 事実：確認できたもの
- 解釈：事実から読み取ったもの
- 仮説：まだ検証していないもの
- 提案：今後行うべきこと
- 演出：理解や印象を補助するもの

すべてを同じ確信度、同じ見た目で扱わない。

## 一貫性と均質性を区別する

書体、色、余白、表記、出典形式は一貫させる。文字量、構図、見出しの強さ、項目数、結論の調子は内容に応じて変える。

## 秩序の上に逸脱を置く

独自性やダイナミックさを得るために、最初からランダムに崩さない。

1. グリッド、文字階層、余白、出典などの秩序を作る
2. 内容に結びついた一つの逸脱ルールを決める
3. 静、標準、密、動の速度差を作る
4. 題材固有の原材料を中心に置く

例：予想が外れる場面だけグリッドから要素を外す、反証ページだけ左右を反転する、同じ引用を意味を変えて再登場させる。

逸脱は、変化をつけるためではなく、内容上の転換を可視化するために使う。

詳細は [`reference-strategy.md`](reference-strategy.md) を参照する。

## 参照先から外見ではなく操作を借りる

参考資料を「この感じで」と模倣しない。

参照先を次の役割に分ける。

- 骨格：出版物、グリッド、組版、階層
- 逸脱：実験的タイポグラフィ、ポスター、コラージュ
- 時間：漫画、映画タイトル、映像、展示動線
- 固有素材：元記事、会話、写真、表、手書き、現場の記録

各参照について、借りる操作、借りない外見、内容との接続、リスクを明示する。

## 媒体に合わせる

- ウェブ：読み手が自分の速度で選択する
- スライド：発表者が時間軸を制御する
- 文書：単独で読んでも理解できる
- SNS：文脈が切り取られることを前提にする

別媒体の見た目だけを移植しない。

## 反証と限界を結論の近くに置く

最後に小さく免責を書くのではなく、主張、根拠、言える範囲、言えない範囲を近接させる。

## AIの役割を限定する

AIへ任せやすいのは候補、抜け、校正、形式、比較軸、反論、体裁。

人間が保持するのは目的、読み手、原材料、優先順位、結論、採否、責任。

AIへ結論、構成、表現、評価を同時に任せない。対話できる場合は重要なステージゲートで一度止め、ユーザーの判断を得てから次へ進む。
