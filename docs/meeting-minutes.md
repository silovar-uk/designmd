# 議事録

> 状態：stable  
> designmd版：2026.07.18-beta  
> 最終更新：2026-07-18

議事録は、会議を魅力的に見せる文章ではない。会議後に、決定、保留、担当、期限、根拠を確認し、仕事を進めるための記録である。

## 1. 見出しの絶対条件

- 見出しは、議題や記載内容を直接示す日本語にする
- あおり文、キャッチコピー、比喩的な見出しを使わない
- 英語見出し、英語副題、日本語見出しに重ねる英語ラベルを使わない
- 重要度は、言葉を強くするのではなく、順序、見出し階層、区分表示で示す

使用できる見出しの例：

- 券売
- 広告
- イベント
- 商品企画・販売
- 特設ページ・情報設計
- 協賛
- ホームタウン
- 決定事項
- 対応事項
- 確認が必要な事項

使用しない見出しの例：

- いま動かなければ、すべてが遅れる
- 次の一手が未来を変える
- 売るのではなく、選ばれる
- 英語だけの見出しや副題

## 2. 基本構成

会議の目的に合わせて、次のいずれかを選ぶ。

### 決定・対応中心

- 会議概要
- 決定事項
- 対応事項
- 議論の背景
- 確認が必要な事項

緊急対応、経営判断、進行管理を優先する会議に向く。

### 議題別

- 券売
- 広告
- イベント
- 商品企画・販売
- 特設ページ・情報設計
- 協賛
- ホームタウン

複数部署の進捗と議論を、テーマごとに確認する会議に向く。各議題の中で「状況」「決定」「方針」「検討」「対応」を分け、共有事項と重要事項を混ぜない。

### 時系列

発言の経緯そのものが重要な場合に限る。通常の実務会議では、画面共有、言い直し、相づちまで残りやすいため、第一候補にしない。

## 3. 残す情報

- 決定したこと
- 決定していないこと
- 判断理由
- 数字、日時、期限
- 担当と関係部署
- 制約、反対意見、実施条件
- 次回に判断すること

発言順を変えてもよい。ただし、誰の意見かが判断に影響する場合は発言者を残す。

## 4. 削る情報

- 相づち
- 同じ内容の繰り返し
- 画面共有や資料探索の操作会話
- 言い直しだけの発言
- 結論に影響しない雑談
- 内容を増やさない予告、総括、あおり文

削ることで判断理由や意見の違いが消える場合は残す。

## 5. 自動文字起こしの扱い

- 固有名詞、金額、日付、人数は推測で補完しない
- 聞き取れない箇所は「要確認」とする
- 正式な数値は、会議資料や管理表と照合する
- 発言から担当者を推測して確定しない
- 認識誤りが多い場合は、発言者名より部署・役割で記載する

## 6. 対応事項

対応事項には、次を付ける。

- 対応内容
- 担当
- 期限
- 状態
- 完了条件

担当または期限が決まっていない場合は、空欄にせず「要確認」「要設定」と記載する。

## 7. 最終確認

- 見出しだけで議題と内容が分かるか
- あおり文、キャッチコピー、英語見出し、英語副題が残っていないか
- 決定、方針、検討、確認が混ざっていないか
- 数字と期限が正式資料と一致しているか
- 担当と完了条件が分かるか
- 自動文字起こしの誤りを推測で埋めていないか
- 次回に何を決めるかが分かるか
