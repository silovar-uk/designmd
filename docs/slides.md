# スライドガイドライン

対象：プレゼン、企画書、社内説明、会議、報告、営業、研究発表、配布資料。

## 1. 使用場面を決める

### 発表用

口頭説明を前提に、1枚の情報量を絞り、視線と時間を制御する。

### 閲覧用

資料単独で理解できるよう、根拠、条件、補足を残す。

### 会議用

判断事項、選択肢、論点、決定、宿題を明確にする。完成品より議論材料を優先する。

### 営業用

相手固有の課題を起点にし、自社紹介を先に置かない。導入後の変化と証拠を示す。

## 2. 完成を一度に作らない

スライド制作は次の順で進める。

1. 逆質問と制作前ブリーフ
2. 原材料と証拠の整理
3. 参照先の役割分け
4. 構成案の比較
5. ストーリーボード
6. グリッドとデザインシステム
7. 代表ページ3〜5枚
8. 全体初稿
9. レビューと削減

構成が未確定の段階で細かな装飾を作り込まない。代表ページで静・標準・密・動が成立する前に、全ページを量産しない。

詳細は [`staged-production.md`](staged-production.md) を参照する。

## 3. AIくさくなりやすい構成

- 左上の英語ラベル
- 大見出し
- 一文要約
- 3枚のカード
- 小さな出典
- 右下のページ番号

この構造が3ページ以上続いたら、内容に合う形式へ変更する。

カードは一覧性に向くが、因果、時間、優先順位、依存関係には向かない。使う前に「本当に並列か」と問う。

## 4. ページの役割

次から選ぶ。

- 観察
- 問い
- 事実
- 比較
- 理論
- 証拠
- 解釈
- 反証
- 判断
- 提案
- 次アクション
- 間
- 転換
- 回収

同じ役割を連続させすぎない。

「一枚一メッセージ」は一枚一断言ではない。問い、比較、未確定事項、複数案、反証、例外、決めるべきことも一つのメッセージである。

## 5. グリッドを関係の文法として使う

グリッドは、同じ箱を繰り返すためのテンプレートではない。

管理するもの：

- 見出しと本文
- 本文と図版
- 図版とキャプション
- 主張と根拠
- ページ間の反復
- 左右、上下の視線移動
- 情報の重要度

### 基本候補

- 6列：大きな構成差を作りやすい
- 12列：2、3、4、6分割へ変換しやすい
- ベースラインまたは垂直方向の基準：本文、注釈、キャプションの呼吸をそろえる

列数は目的ではない。内容に必要な組み合わせから決める。

### 固定するもの

- 安全余白
- 基準線
- 文字階層
- 出典、注釈、ページ番号の位置
- 色の役割

### 変えるもの

- 使用する列数
- 図版の大きさ
- 余白量
- 情報密度
- 視線の方向
- ページの入口

同じグリッドから異なるページを作り、異なるページが同じ世界に属して見える状態を目指す。

## 6. 内容に合うレイアウト

- 引用：原文、発言者、日時、出所、解釈
- 比較：共通の軸、差分
- 因果：原因、中間過程、結果、確認状況
- 時系列：方向、日時、転換点
- 研究：対象、方法、結果、限界
- 提案：現状、問題、対応、効果、リスク、判断事項
- 場面：発言、状況、反応、解釈
- 反証：主張、別の説明、言える範囲の更新

内容の種類によって構造を変える。見た目の変化だけを目的にしない。

## 7. 密度と速度を設計する

全ページを同じ情報量にしない。

### 低密度

- 表紙
- 問い
- 引用
- 章扉
- 結論
- 間

### 標準密度

- 一つの主張と根拠
- 比較
- 一つの図解
- 具体例

### 高密度

- 研究結果
- 表
- 複数の引用
- 反証
- 注釈

資料全体に静、標準、密、動の差を作る。

例：

- 静 → 標準 → 密 → 静 → 動 → 余韻
- 問い → 具体例 → 理論 → 反証 → 再解釈

すべてのページを強くすると、全体は平らになる。

## 8. 秩序の上に一つの逸脱ルールを置く

独自性やダイナミックさを、ランダムな崩しで作らない。

逸脱ルールの例：

- 予想が外れるページだけ要素がグリッドから外れる
- 反証ページだけ左右を反転する
- 本音の発言だけ欄外へ出す
- 転換点だけ一語が画面外へ切れる
- 同じ図を意味を変えて再登場させる

使用できる操作：

- 拡大
- 切断
- 方向
- 反復
- 圧縮
- 重なり
- 反転
- 回収

一ページでは主となる操作を一つに絞る。逸脱には、内容上の理由を持たせる。

## 9. 参照先を役割別に使う

- 骨格：書籍、雑誌、新聞、研究誌、図録
- 逸脱：実験的タイポグラフィ、ポスター、コラージュ
- 時間：漫画、映画タイトル、映像、舞台、展示動線
- 固有素材：元記事、会話、写真、表、手書き、現場記録

各参照について、借りる操作と借りない外見を明示する。

「雑誌風」「映画風」といった雰囲気指定だけで進めない。

詳細は [`reference-strategy.md`](reference-strategy.md) を参照する。

## 10. 代表ページを先に試す

全体制作前に、3〜5ページを作る。

最低限：

1. 静かなページ
2. 標準ページ
3. 密度の高いページ
4. 動きのあるページ

確認する。

- 同じ世界に属して見えるか
- 役割に応じて形が変わっているか
- グリッドがテンプレート化していないか
- 逸脱が内容と結びついているか
- 固有素材が主役になっているか
- ページ間の速度差があるか

対話可能なら、ここでユーザーへ提示し、承認後に全体へ進む。

## 11. 視覚的階層

結論、根拠、補足、出典、注記、例外、次アクションを区別する。大きい文字を置くだけでは階層にならない。

出版物のように、複数の声を持たせる。

- 見出し：論点
- 本文：事実や説明
- 引用：当事者の声
- 注釈：編集者の声
- 出典：検証可能性
- ページ番号／柱：現在位置

すべてをプレゼンターの強い声へ変換しない。

## 12. ページ間の変化

- 理論の次に具体例
- 密度の高いページの次に引用
- 強い主張の次に限界
- 全体像の次に一点拡大
- 数値の次に現場の声
- 同じ素材を後半で再解釈

内容の変化に応じて構成を変える。ランダムに崩さない。

## 13. 英語ラベル

業界で一般的、固有名称、日本語より短く正確、構造上必要、読み手が理解できる場合に限る。「INSIGHT」「CORE VALUE」などを装飾目的で使わない。

## 14. 出典

主張の近くに、発行元、年、資料名、URLまたは文献情報を置く。数値には期間、範囲、単位、比較条件、更新時点を付ける。

## 15. ページごとの確認

1. ページの役割は何か
2. なぜこのレイアウトか
3. グリッドのどの関係を使っているか
4. 他ページと同じ構成でよい理由はあるか
5. 画像や図を消すと理解は落ちるか
6. 事実、解釈、提案が分かれているか
7. 読み手は何を判断すべきか
8. 単独で見ても誤解しないか
9. このページの密度と速度は何か
10. 逸脱がある場合、その内容上の理由は何か
11. 題材固有の情報があるか
12. 前後のページで理解がどう変わるか
