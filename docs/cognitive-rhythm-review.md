## 4. 制作者確認

- [ ] 文書進行だけを実況する文が減った
- [ ] 問いが後の説明・判断を変えた
- [ ] 不要な問い、告白、短文が増えていない
- [ ] 事実、条件、留保が保存された
- [ ] 題材固有のリズムになった
- [ ] 文書種別に応じて適用範囲を変えた
- [ ] 本人の語彙と呼吸で音読確認した

## 5. 失敗パターン

- [ ] 問いから入る文章ばかりになった
- [ ] 意外な事実を無理に探した
- [ ] 書き手が持っていない迷い・告白を足した
- [ ] 結論や重要条件が遅くなった
- [ ] 短い断定文が増えすぎた
- [ ] 節末の予告を消した結果、検索性が落ちた
- [ ] 列挙へ説明を足しすぎ、冗長になった
- [ ] 認知モード分類に時間を使いすぎた
- [ ] 読み進めやすさは上がったが、理解が落ちた
- [ ] 文書ごとの差が減り、同じ型になった

## 6. 修正判断

### 維持

- 内容理解と事実保存が保たれている
- 問い、告白、短い断定が定型化していない
- 文書種別に応じて適用範囲を変えられている

### 縮小

次の場合は、対象更新テストと問い・回収台帳だけを残す。

- 認知モード分類や密度波形の負荷が高い
- 演出が内容より前へ出る
- 文書が同じ型へ収束する

### 前工程へ戻る

- 理解が落ちた：論証・構成へ戻る
- 結論が遅れた：媒体・使用場面へ戻る
- 書き手にない経験や感情が増えた：原材料・本人確認へ戻る
- 読みづらい：文型、音読、段落分割へ戻る
