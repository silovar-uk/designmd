# 読者向け成果物・学習資料レビュー

各項目を0（問題なし）、1（少し該当）、2（明確に該当）で確認します。

## 1. 本編と制作工程の分離

- [ ] 制作ブリーフが本編へ掲載されている
- [ ] 読み手、使用場面、制作意図が、内容理解に不要なのに表示されている
- [ ] `QUESTION`、`FACT`、`CAUSE` などの内部タグが装飾として表示されている
- [ ] 構成案、採用理由、不採用理由が本編を圧迫している
- [ ] 「この資料の読み方」が、内容そのものより長い
- [ ] 制作判断を残すことと、読者へ見せることを混同している
- [ ] 台本、付録、制作ログの置き場所が分かれていない

### 危険信号

> この情報は、読者のためではなく、制作者がガイドを守った証明になっていないか。

---

## 2. 旧版からの機能保存

- [ ] 旧版の各ページが担っていた理解上の機能を洗い出していない
- [ ] 削除したページの機能を、新版のどこが引き継ぐか不明
- [ ] 手順、具体例、変化、失敗例がデザイン整理の過程で減っている
- [ ] 見た目以外に改善した点を説明できない
- [ ] 新版で失われた情報を記録していない
- [ ] 旧版と新版を同じ閲覧条件で比較していない
- [ ] 「情報量を減らした」ことを、自動的に改善と判断している

### 必須記録

| 旧版の要素 | 機能 | 新版の担当 | 保存／弱化／消失 |
|---|---|---|---|

---

## 3. 学習機能

- [ ] 何を学ぶ資料か、冒頭で分からない
- [ ] 定義がない
- [ ] どの場面が該当するか識別できない
- [ ] なぜ成立するか具体的に説明していない
- [ ] 手順を正しい順番で再現できない
- [ ] 相手や状況が変わった場合の主要な変化がない
- [ ] 失敗条件、反例、撤退条件がない
- [ ] 自分で判断するチェック項目がない
- [ ] 結論が抽象的なコピーで、実戦ルールになっていない
- [ ] 資料を閉じた後に使える知識になっていない

---

## 4. 具体性

- [ ] 「意味」「構造」「因果」「価値」「変換」などの抽象語だけで説明が終わる
- [ ] 抽象語の近くに、数字、実物、操作、発言、Before／Afterがない
- [ ] 初心者に全体図を見せるだけで、注目箇所を限定していない
- [ ] 具体例が1つもない
- [ ] 成功例だけで、境界を示す失敗例がない
- [ ] 主要な応答や分岐が省略されている
- [ ] 読者が判断できない言葉をチェック項目に使っている

---

## 5. 図と盤面・画面

- [ ] 図が本文を箱へ入れ直しただけ
- [ ] 本文を隠すと、図から新しい関係を読み取れない
- [ ] 「2つだけ見る」と書きながら、全要素が同じ濃度で表示されている
- [ ] 注目箇所の拡大、減光、マーキングがない
- [ ] 手順を一枚へ詰め込み、変化を追えない
- [ ] Before／Afterで位置や比較軸が変わっている
- [ ] 矢印が表す動詞を説明できない
- [ ] 図が証拠ではなく、雰囲気として置かれている

---

## 6. ページ役割

- [ ] ページ役割が、ラベルを読まないと分からない
- [ ] 内部設計用の英語ラベルを、読者向けナビゲーションとして使っている
- [ ] ラベルを削除するとページの目的が分からなくなる
- [ ] 読者に必要な日本語見出しより、制作者用語を優先している
- [ ] 役割タグの表示が、ページ間の単調さを強めている

### テスト

役割ラベルをすべて隠し、それでも各ページの機能が伝わるか確認します。

---

## 7. メタ情報の量

- [ ] 本編に制作ブリーフが1ページ以上ある
- [ ] 本編に制作判断ログがある
- [ ] 読む順番の説明が、内容の持ち帰りより目立つ
- [ ] 出典と制作ログが同じページに混在している
- [ ] 付録へ移せる比較や詳細が本編を圧迫している
- [ ] 制作者の反省やガイド適用が、題材の説明より目立つ

---

## 8. 代表ページの理解テスト

- [ ] 30秒見ても、何のページか説明できない
- [ ] どこから見ればよいか分からない
- [ ] ページを閉じた後、一文で再話できない
- [ ] 手順を見ずに並べ直せない
- [ ] 新しい例に対して、使える／使えないを判断できない
- [ ] 本文を隠すと図の意味が分からない
- [ ] 図を隠すと論理が通らない
- [ ] 旧版より理解が速くなったか確認していない

---

## 9. 実寸閲覧

- [ ] スマートフォン1ページ表示で主要図を判別できない
- [ ] PC 50％表示で本文や注釈を読めない
- [ ] A4縮小印刷で図表が潰れる
- [ ] 3秒で視線の入口を見つけられない
- [ ] 拡大しないと駒、記号、数字が判別できない
- [ ] 一覧表示の美しさを、単ページの理解より優先している
- [ ] 余白が、焦点ではなく単なる空白になっている

---

## 10. 台本

- [ ] 台本が本編に表示されている
- [ ] 台本がノート欄または別ファイルにない
- [ ] 台本がスライド本文の読み上げになっている
- [ ] 台本で説明することを理由に、図から重要な関係を省いている
- [ ] 台本で一度も触れない図形や矢印が残っている
- [ ] 前後ページへの接続がない

---

## 11. 戻る段階

- メタ情報が本編を圧迫：成果物の4層分離へ戻る
- 旧版の機能が消失：再設計前の機能保存表へ戻る
- 定義・手順・失敗が不足：学習構成へ戻る
- 抽象語が多い：原材料と具体例へ戻る
- 図が本文の再掲：図の役割と具体物へ戻る
- 実寸で読めない：レイアウト、図の範囲、文字サイズへ戻る
- 新版が旧版より教えにくい：装飾ではなく構成へ戻る

---

## 12. 最終12問

1. 本編に残した情報は、本当に読者へ必要か
2. 台本、付録、制作ログへ移すべきものはないか
3. 旧版の各機能を、新版のどこが引き継いだか
4. 新版で失われた機能は何か
5. 読者は何を定義できるようになるか
6. 何を見て該当場面を識別するか
7. 何をどの順番で行うか
8. 状況が変わったらどう対応するか
9. いつ使わないか
10. 抽象語の近くにどの具体的証拠があるか
11. 実寸で図と注釈を読めるか
12. 新版は旧版より、理解・判断・行動のどれを改善したか
