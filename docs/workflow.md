# 制作ワークフロー

## 避ける順序

1. テイストを決める
2. AIに構成を作らせる
3. AIに文章を書かせる
4. AIに画像を作らせる
5. AIに評価させる
6. 人間が誤字だけ直す

## 推奨順序

### 1. 目的

- 読み手
- 使用場面
- 読み手にしてほしい判断
- 読後に残したい内容
- 読み手が現在知っていること
- 誤解しやすいこと
- 使用時間
- 提出形式

「分かりやすく伝える」を目的にしない。

### 2. 原材料

原文、会話、データ、日時、場所、写真、画面、既存資料、研究、反対意見、失敗、制約、読み手の質問を集める。この段階ではきれいにまとめない。

### 3. 分類

事実、解釈、未確定、提案へ分ける。

### 4. 人間が核を選ぶ

一番重要な事実、意外だったこと、誤解される点、削れるもの、削ると別の話になるもの、納得していない点、断定可能範囲を決める。

### 5. AIに候補を出させる

別解釈、反論、抜けている関係者、読み手別の疑問、異なる構成案、事実と解釈の混同を出させる。採用前提にしない。

### 6. 構成

文章の定型へ固定せず、スライドは各ページの機能を割り当てる。構成理由を一文で説明する。

### 7. 初稿

装飾を後回しにする。白背景、標準フォント、最小限の色、仮画像、仮図、出典優先で論理を確認する。

### 8. 批評

抽象語、二項対立、構文、具体性、根拠、反証、文化的違和感、媒体不整合、テンプレート反復、装飾の意味を指定して確認する。

### 9. 棄却

AI案を採用、一部採用、不採用、保留へ分ける。一般論、相手不在、根拠不足、重複、過剰断定、見た目だけ、情報が増えない、など不採用理由を残す。

### 10. 最終確認

声に出す、白黒で見る、画像を消す、見出しだけ読む、本文だけ読む、出典だけ確認する、順番を変える、他テーマへ転用できる文を探す。

## 制作ログ

```text
目的：
読み手：
採用した案：
採用理由：
捨てた案：
不採用理由：
未確認事項：
最終判断者：
```
