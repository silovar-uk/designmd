# 制作ワークフロー

## 避ける順序

1. テイストを決める
2. AIに完成構成を作らせる
3. AIに文章を書かせる
4. AIに全ページをデザインさせる
5. AIに画像を作らせる
6. AIに自己評価させる
7. 人間が誤字だけ直す

この順序では、最初に選ばれた無難な方向が、文章、構成、レイアウト、画像へ連鎖する。

## 推奨する考え方

完成品を一度に作らない。

> 各段階で、次の段階へ進むために必要な判断だけを確定する。

詳細な通過条件は [`staged-production.md`](staged-production.md) を参照する。

# 推奨順序

## GATE 0：目的と不足情報

確認する。

- 読み手
- 使用場面
- 読み手にしてほしい判断、理解、行動
- 読後に残したい内容
- 読み手が現在知っていること
- 誤解しやすいこと
- 使用時間
- 提出形式
- 固定したいもの
- 違和感を許せる場所
- ありきたりだと感じる表現

不足がある場合は、最大7〜8問に絞って逆質問する。

「分かりやすく伝える」だけを目的にしない。

## GATE 1：原材料と証拠

原文、会話、データ、日時、場所、写真、画面、既存資料、研究、反対意見、失敗、制約、読み手の質問を集める。

この段階ではきれいにまとめない。

次へ分類する。

- 事実
- 解釈
- 仮説
- 未確定
- 提案
- 演出候補

この制作物でしか使えない発言、数字、場面、失敗、違和感を特定する。

## GATE 2：参照戦略

参照資料を四役へ分ける。

1. 骨格：出版物、グリッド、組版、階層
2. 逸脱：実験的タイポグラフィ、ポスター、コラージュ
3. 時間：漫画、映画タイトル、映像、展示動線
4. 固有素材：元記事、会話、写真、表、手書き、現場記録

各参照について、借りる操作、借りない外見、内容との接続、リスクを記録する。

詳細は [`reference-strategy.md`](reference-strategy.md) を参照する。

## GATE 3：構成仮説

最低3案を比較する。

例：

- 具体的な場面から始める
- データや研究から始める
- 誤解や反論から始める
- 同じ素材を二つの見方で読み直す

各案について、強み、弱み、使用場面との相性、面白味が生まれる場所、誤解のリスク、不採用理由を示す。

「最もきれいな案」ではなく、目的と原材料に最も合う案を選ぶ。

## GATE 4：内容の設計図

文章の定型へ固定しない。

スライドは、各ページへ次の機能を割り当てる。

- 観察
- 問い
- 事実
- 比較
- 理論
- 証拠
- 解釈
- 反証
- 判断
- 提案
- 次アクション
- 間
- 転換
- 回収

構成理由、原材料との対応、情報を明かす順序、密度と速度を決める。

この段階では、完成レイアウトを作らない。

## GATE 5：デザインシステム

全ページを作る前に、次を決める。

### 秩序

- グリッド
- 垂直方向の基準
- 書体と文字階層
- 余白の単位
- 出典、注釈、ページ番号の位置
- 色の役割

### 変化

- 低密度、標準密度、高密度
- 全体の速度
- 繰り返すモチーフ
- 一つの逸脱ルール

逸脱は、内容上の転換と結びつける。

## GATE 6：代表ページ

全体を量産する前に、3〜5ページを試作する。

最低限、次を含める。

1. 静かなページ
2. 標準ページ
3. 密度の高いページ
4. 動きのあるページ

単独の完成度だけでなく、ページ間の変化を確認する。

確認項目：

- 同じ世界に属して見えるか
- 役割に応じて形が変わっているか
- 逸脱が内容と結びついているか
- 読む速度に差があるか
- 固有素材が中心になっているか

対話可能なら、この段階でユーザーの判断を得る。承認前に全ページを量産しない。

## GATE 7：全体初稿

代表ページで確認したルールを使って全体を制作する。

ここで新しいスタイルを次々に追加しない。

必要な例外は、内容上の理由とともに記録する。

## GATE 8：批評と棄却

評価軸を指定して確認する。

- 抽象語
- 二項対立
- 構文の反復
- 具体性
- 根拠
- 反証
- 文化的違和感
- 媒体不整合
- テンプレート反復
- 装飾の意味
- 早すぎる収束
- 参照先の表面的模倣
- 逸脱の理由
- 密度と速度

AI案を、採用、一部採用、不採用、保留へ分ける。

一般論、相手不在、根拠不足、重複、過剰断定、見た目だけ、情報が増えない、など不採用理由を残す。

## GATE 9：最終確認

- 声に出す
- 白黒で見る
- 画像を消す
- 見出しだけ読む
- 本文だけ読む
- 出典だけ確認する
- ページを一覧表示する
- 順番を変える
- 低密度／高密度だけを抜き出す
- 別テーマへ転用できる文や素材を探す
- 逸脱箇所の理由を説明する

# 対話モード

ユーザーとのやり取りが可能な場合は、原則として次で一度止める。

1. GATE 0：逆質問
2. GATE 3：構成案の選択
3. GATE 6：代表ページの確認

ユーザーが承認する前に、完成品を量産しない。

# 一括モード

一度の出力を求められた場合も、内部では全ゲートを順番に通す。

提出時に、次を示す。

- 置いた前提
- 比較した構成案
- 採用理由
- デザインシステム
- 逸脱ルール
- 代表ページで確認したこと
- 捨てた案
- 未確認事項

一括モードは、工程を省略するモードではない。確認を内部化するモードである。
