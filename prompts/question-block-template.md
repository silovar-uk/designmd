# 逆質問テキストブロック用プロンプト

制作前に不足情報を確認するとき、通常の箇条書きではなく、次の形式で返してください。

```text
【こちらで置いた前提】
- テーマ：
- 成果物：
- 既に分かっている要望：

【必須】
1. 読み手：
   回答：

2. 使用場面：
   回答：

3. 読後に求める理解・判断・行動：
   回答：

【推奨】
4. 今回いちばん出したい温度：
   選択肢：創業者の原体験／プロダクトローンチ／実験発表／チームのキックオフ／未来像
   回答：

5. ユーモアの強さ：
   選択肢：静かな微笑み／軽いツッコミ／自虐を含む笑い／大胆な実験
   回答：

6. 実例や失敗の扱い：
   選択肢：実物を見せる／再現図にする／文章だけで触れる／出さない
   回答：

【任意】
7. 参考にしたい媒体や雰囲気：
   回答：

8. 絶対に避けたい表現：
   回答：
```

ルール：

- 既に分かっていることは、質問せず「こちらで置いた前提」に入れる。
- 必須、推奨、任意の三分類にする。
- 質問は最大8問。
- 各質問に `回答：` を付ける。
- 回答前に完成版の制作へ進まない。
- ユーザーが「質問不要」「一括で」と明示した場合のみ、質問を省略する。
