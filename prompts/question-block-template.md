# 最初の一問をつくるプロンプト

DesignMDを使う制作では、計画、調査、構成、制作へ入る前に、必ず一問だけ質問してください。

質問の目的は、不足情報をすべて集めることではありません。回答によって成果物が最も大きく変わる判断を、一つだけ共有することです。

## 手順

1. 会話、添付資料、過去のフィードバックから、すでに確定していることを内部で整理する。
2. 未確定事項を複数考える。
3. 構成、役割、WAO、調査範囲、納品方法を最も変えるものを一つだけ選ぶ。
4. 今回の題材に合う具体案を2〜3個出す。
5. 推奨案と短い理由を添える。
6. 最初の応答では、その一問だけを提示する。
7. 回答後は追加質問をせず、残る不足を仮説、未確認事項、制作範囲として管理する。

## 出力形式

```text
最初に一つだけ確認します。
今回、［成果物を最も変える選択］は A｜［具体案］、B｜［具体案］、C｜［具体案］のどれにしますか？ 推奨は［案］です。理由は［短い理由］です。
```

## 選ぶ優先順位

1. 安全、法令、権利、公開可否に関わる必須事実
2. 読み手と使用場面
3. 読後に変えたい理解、感情、判断、行動
4. 採用する問い、中心メッセージ
5. WAOで上振れさせたい期待
6. 編集型、資料モード、表現の強度
7. 色、書体、細かな好み

## ルール

- 最初の実質的な応答を質問にする。
- 質問は一問だけにする。
- 疑問文と質問符は一つだけにする。
- 一文へ複数の確認事項を詰め込まない。
- 既に分かっていることを聞き直さない。
- 選択肢だけを並べず、推奨案と理由を示す。
- 記号、一語、短い修正で答えられる形にする。
- 質問前に計画、分析、制作案を出さない。
- 回答前に完成版の制作へ進まない。
- 回答後に追加質問をしない。
- ユーザーが「質問不要」「一括で」と指定しても、最初の一問は省略しない。
- 安全、法令、権利の必須事実が欠ける場合は、最初の一問をその確認に使う。
- 残る不足は、置いた前提、未確認事項、制作範囲から外すものとして明示する。

## 回答後の記録

```text
【最初の一問】
質問：
回答：
回答によって変えた制作条件：

【置いた前提】
- 

【未確認事項】
- 

【今回の制作範囲から外すもの】
- 
```
