# Wordマニュアルを作成する

以下を、そのままAIへ渡して使う。

```text
以下のガイドを必須要件として参照してください。

https://raw.githubusercontent.com/silovar-uk/designmd/main/docs/manual.md
https://raw.githubusercontent.com/silovar-uk/designmd/main/docs/font-policy.md
https://raw.githubusercontent.com/silovar-uk/designmd/main/docs/writing.md

今回の依頼：
提供する原材料から、実作業で使えるWordマニュアルを作成してください。

最初の応答では、成果物を最も大きく変える一問だけを聞いてください。
既に分かっていることは聞き直さず、具体的な選択肢と推奨案を添えてください。
回答後は追加質問をせず、残る不足は未確認事項として管理してください。

必須要件：
- 一般的なベストプラクティスを、確認なしで現行手順へ追加しない
- 確認済み手順、実際の事故、例外、担当、リンク、未確認事項、改善案を分ける
- 組織分類ではなく、担当者が実際に作業する順に並べる
- 分類名より、何がどう間違うかを具体的に書く
- 危険度、難易度、事故頻度に応じて説明量へ差を付ける
- 同じ情報を本文、表、チェックリストで繰り返さない
- 部署や担当領域を責任主体、個人名を現在の窓口として使い分ける
- 例外には、確認できる過去事例と採用理由を残す
- リンク、連絡先、ファイルは実際に使うSTEPの近くへ置く
- 現場のコメントが示す重要度や判断を、一般論へ丸めない
- 不明な手順、担当、日付、承認、保存ルールを推測で補わない

Wordデザイン：
- 日本語フォントはBIZ UDPゴシックに固定する
- 「標準」、見出し1〜3、箇条書き、表、キャプション、ヘッダー、フッターへBIZ UDPゴシックを適用する
- 指定フォントがない場合、游ゴシックやメイリオへ黙って代替しない
- 白地、黒または濃いグレーの文字を基本にする
- 本文は10.5〜11ptを基本にし、既存テンプレートがある場合はサイズや余白を調整してよい
- 正式テンプレートに変更禁止のフォント規定がある場合だけ、その規定を優先する
- Wordの組み込み見出しスタイルを使用する
- 赤は重大事故、禁止、必須確認、高リスク設定だけに使う
- 赤文字には「重要」「注意」「禁止」などの文言を併記し、色だけに意味を依存させない
- 一つの節に主となる強調は原則一つにする
- 太字、赤字、黄色マーカー、網掛け、囲みを一か所へ重ねない
- 表は日付、設定値、役割、状態など同じ軸の比較にだけ使う
- スクリーンショットは操作対象が読める大きさに切り取り、番号を付けすぎない
- 見出しだけをページ末尾へ残さない
- リンクをクリック可能にし、用途が分かる表示名にする
- 必要な画像へキャプションまたは代替説明を付ける

制作順：
1. 原材料を、確認済み手順／事故／担当／例外／参照先／未確認／改善案へ分ける
2. 実作業順の構成を作る
3. 危険箇所だけ説明を濃くする
4. 重複を削る
5. Wordの見出し、改ページ、表、画像、リンク、強調を整える
6. BIZ UDPゴシックがスタイルと個別文字へ適用されているか確認する
7. 実寸、白黒、リンク、ナビゲーション、アクセシビリティを確認する

原材料：

```
