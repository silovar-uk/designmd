# 問いの監査プロンプト

制作へ入る前に、与えられたブリーフの前提をAIと人間で監査する。

```text
以下のガイドを参照してください。

共通原則：
https://raw.githubusercontent.com/silovar-uk/designmd/main/DESIGN.md

問いの監査：
https://raw.githubusercontent.com/silovar-uk/designmd/main/docs/question-audit.md

# 目的

与えられた問いへすぐ答えず、前提、根拠、本当に変えたい状態、外部文脈、代替問いを確認してください。

AIは問いを問い直して構いません。ただし、奇抜な問いや反転案を出すこと自体を目的にしないでください。採用する問いの最終判断は人間が行います。

# 問いの監査

1. 前提
   ブリーフが事実として扱っている内容を抽出してください。

2. 根拠
   各前提を、確認済み事実／依頼者の仮説／慣習的な表現／未確認に分類してください。

3. 欲望
   表面的な課題が解決した先で、依頼者は何を変えたいと考えられるか、複数仮説を示してください。断定しないでください。

4. 文脈
   生活者、競合、組織、社会、時代の観点から、不足している材料を示してください。確認できない事実を創作しないでください。

5. 代替問い
   次の三案を作ってください。
   - ブリーフ忠実案
   - 欲望へ持ち上げた再定義案
   - 前提を採用しない反転・別軸案

# 比較

三案を、次の観点で比較してください。

- 根拠
- 得られるもの
- 失うもの
- 実装可能性
- 誤解・倫理・組織上のリスク

# 禁止

- 「常識を疑う」「逆転の発想」などの定型だけで問いを変えない
- 社会課題へ接続することを自動的に上位概念としない
- 根拠なく「本当の課題は別にある」と断定しない
- 依頼者の目的を理解せず、ブリーフを否定しない
- 緊急、安全、法令、決定済み事項を面白さのために広げない

# 出力

1. 依頼された問い
2. 前提と根拠の状態
3. 本当に変えたい状態の仮説
4. 不足している外部文脈
5. 三つの代替問い
6. 比較表
7. 推奨する問いと理由
8. 採用時に必要な追加調査
9. 人間が最終判断すべき点

ブリーフ：
（ここに貼る）
```
