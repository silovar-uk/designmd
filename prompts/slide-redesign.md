# 既存スライド再設計プロンプト

既存のPowerPoint／PDFを作り直すときに使用します。

```text
以下のガイドを必須要件として参照してください。

共通原則：
https://raw.githubusercontent.com/silovar-uk/designmd/main/DESIGN.md

スライド設計：
https://raw.githubusercontent.com/silovar-uk/designmd/main/docs/slides.md

読者向け成果物と制作ログ：
https://raw.githubusercontent.com/silovar-uk/designmd/main/docs/audience-first-redesign.md

学習資料：
https://raw.githubusercontent.com/silovar-uk/designmd/main/docs/learning-materials.md

図形・矢印・台本：
https://raw.githubusercontent.com/silovar-uk/designmd/main/docs/diagram-shapes.md

専用レビュー：
https://raw.githubusercontent.com/silovar-uk/designmd/main/docs/audience-learning-review-checklist.md

# 最重要方針

旧版を捨てて、いきなり新しいデザインへ置き換えないでください。

先に、旧版が読者へ提供していた理解上の機能を洗い出してください。

制作ブリーフ、ページ役割、構成案、採用理由などの内部情報は制作ログへ残し、読者に必要な理由がない限り本編へ表示しないでください。

# GATE 0：使用場面

次を確認してください。

- 読み手
- 発表用／閲覧用／会議用／学習用
- 読後に求める理解、判断、行動
- 閲覧端末
- 印刷の有無
- 旧版について残したい点、問題点

# GATE 1：旧版の機能保存表

旧版の各ページまたは要素について、次の表を作ってください。

| 旧版の要素 | 読者に与えていた機能 | 削除した場合の損失 | 新版の担当箇所 | 保存状況 |
|---|---|---|---|---|

機能の候補：

- 定義
- 識別
- 理由
- 手順
- 具体例
- 変化
- 失敗条件
- 判断基準
- 記憶の手掛かり

新版に担当箇所がない場合、「整理」ではなく「機能消失」と記録してください。

対話可能な場合は、ここで一度止まってください。

# GATE 2：成果物の4層

情報を次へ分けてください。

A. 読者向け本編
B. 発表者用台本
C. 付録・根拠
D. 制作ログ

次は原則として制作ログへ置いてください。

- 制作ブリーフ
- ページ役割タグ
- 構成案
- 参照戦略
- 採用・不採用理由
- ガイド適用の記録

次の問いに「はい」と答えられる情報だけを本編へ置いてください。

「この情報を読者が知らなければ、内容の理解、判断、行動が難しくなるか」

# GATE 3：学習機能

学習・解説資料の場合、次がどこにあるか示してください。

1. 定義
2. 識別
3. 成立理由
4. 手順
5. 主要な変化
6. 失敗条件
7. 自分で判断する基準

欠けているものがある場合、見た目を作る前に構成を修正してください。

# GATE 4：抽象語と具体物

「意味」「構造」「因果」「価値」「変換」「成立条件」などの抽象語を抽出してください。

各抽象語について、近くに置く具体的証拠を示してください。

- 数字
- 発言
- 実物
- 操作
- Before／After
- 拡大図
- 主要な応答
- 失敗例

抽象語だけで説明を終えないでください。

# GATE 5：構成案

最低3案を比較してください。

各案について、次を示してください。

- 旧版から保存する機能
- 新たに改善する機能
- 失う可能性がある機能
- 初心者がつまずく場所
- 本編から制作ログへ移す情報
- 付録へ移す情報

対話可能な場合は、ここで一度止まってください。

# GATE 6：ストーリーボードと台本

各ページについて示してください。

- 読者向けの役割
- 読者向け見出し
- 使用する具体物
- 学習機能
- 本編に表示する情報
- ノート欄へ入れる文章形式の台本
- 付録へ送る情報
- 制作ログへ残す情報

内部役割の `QUESTION`、`FACT`、`CAUSE`、`REFUTE` などを、読者に必要な理由なく表示しないでください。

# GATE 7：代表ページ

最低限、次を試作してください。

- 定義または基本形
- 手順
- 具体例または拡大図
- 主要な変化
- 失敗条件
- 判断チェック

各ページで次を実施してください。

- 30秒テスト
- 再話テスト
- 再現テスト
- 図単独テスト
- 本文単独テスト
- 旧版比較テスト

デザイン上の統一感だけで通過させないでください。

対話可能な場合は、ここで一度止まってください。

# GATE 8：実寸閲覧

次の条件で確認してください。

- スマートフォン縦画面で1ページ表示
- PC 50％表示
- A4縮小印刷
- 3秒で視線の入口が分かるか
- 図表、盤面、注釈を拡大せず判別できるか

一覧画像の美しさより、単ページの理解を優先してください。

# GATE 9：全体制作

代表ページで確認した構造を展開してください。

PowerPointでは、各スライドのノート欄へ文章形式の台本を入れてください。

台本は本編へ表示せず、スライド上の情報をそのまま読み上げる内容にしないでください。

# 最終提出

次を添えてください。

- 旧版の機能保存表
- 本編／台本／付録／制作ログの分離表
- 新版で改善した理解・判断・行動
- 新版で弱化または消失した機能
- 学習機能の配置
- 代表ページの理解テスト結果
- 実寸閲覧テスト結果
- 各ページの台本
- 制作ログ
```
