# スライドタイトル監査プロンプト

以下のスライド構成または既存タイトル一覧を監査してください。

## 目的

資料の目的と各ページの役割に応じて、タイトルの強度を調整する。

タイトルを単に短く、おとなしくするのではなく、次を明確にする。

- 何についてのページか
- 何を示すページか
- 何を読み取るページか
- 何を提案・判断するページか

## 手順

1. 資料全体の主モードを判定する
   - 説明・報告
   - 説得・提案
   - 読み物・体験
   - 混合

2. 各ページの役割を判定する
   - 観察・事実・証拠
   - 比較・分析
   - 解釈
   - 判断・提案
   - 次アクション
   - 間・転換・章扉

3. 各タイトルの強度を判定する
   - レベル1：内容説明型
   - レベル2：解釈併記型
   - レベル3：提案・判断型
   - コピー型：章扉・転換用

4. 次の問題を検出する
   - 事実ページが根拠以上の断言になっている
   - 提案ページが「概要」「について」だけで終わっている
   - すべてのタイトルがキャッチコピーになっている
   - 「AではなくB」、比喩、命令形、短い断言が連続する
   - タイトルだけでは対象と内容を予測できない
   - タイトルと副題が同じ意味を繰り返す

5. 内容説明型の仮タイトルを一度作る

6. 解釈または提案が必要なページだけ、適切な強度へ上げる

7. 全タイトルを一覧し、事実から提案、次アクションまで論理が進むか確認する

## 出力形式

### 1. 資料モード

- 主モード：
- 従モード：
- 判定理由：

### 2. 全体所見

- 現在のタイトル設計の問題：
- 推奨するタイトル設計：
- 強いタイトルを残すページ：

### 3. ページ別修正

各ページについて次を示す。

```text
ページ番号：
ページ役割：
現行タイトル：
現行強度：
問題：
修正タイトル：
修正強度：
修正理由：
```

### 4. 修正後タイトル一覧

タイトルだけをページ順に並べる。

### 5. 最終確認

- タイトルだけで各ページの内容が分かるか
- 事実から解釈、提案へ論理が進むか
- 強いタイトルが必要なページへ限定されているか
- 同じ言い回しが連続していないか

## 制約

- 刺激的であることを採用理由にしない
- 元資料にない意味、感情、因果を創作しない
- 数字や固有名詞を、抽象的なコピーへ丸めない
- 提案資料でも、事実ページは説明型を基本とする
- 修正案は、タイトルだけを読んでも対象と内容が分かる形を優先する
