# 文章レビュー用プロンプト

```text
以下の文章を、内容の正しさに加え、「誰も本気で選んでいないように見える文章」になっていないか点検してください。

共通原則：
https://raw.githubusercontent.com/silovar-uk/designmd/main/DESIGN.md

文章ガイド：
https://raw.githubusercontent.com/silovar-uk/designmd/main/docs/writing.md

日本語のAIくささを減らす実践編集：
https://raw.githubusercontent.com/silovar-uk/designmd/main/docs/japanese-ai-writing-practices.md

専用チェックリスト：
https://raw.githubusercontent.com/silovar-uk/designmd/main/docs/japanese-writing-review-checklist.md

# 最重要方針

禁止語を別の語へ置換するだけで終えないでください。
文章を、次の3回に分けて点検してください。

## PASS 1：表面とリズム

- 不自然な記号、半角スペース、Markdown記法
- 同じ文末、接続詞、文長の反復
- 内容を増やさない予告、総括、姿勢表明
- 空虚な形容詞と動詞
- 見出しや箇条書きまで、すべて完全な文になっていないか
- 同じリスト内で文型が無目的に混ざっていないか

この段階では、内容を勝手に増やさないでください。

## PASS 2：段落と論証

各段落について、次を一文ずつ示してください。

- 前段落から何を受けているか
- この段落が何をしているか
- 次段落へ何を渡しているか

そのうえで、次を確認してください。

- 同じ主張の言い換え
- 因果の仕組みの欠如
- 主張と証拠の不一致
- 複数要因の単純化
- 理論や引用の見せびらかし
- 話題の割り込み
- 条件と限界の不足
- 確認していない内容の補完

必要なら、文の修正だけでなく、段落の削除、統合、移動を提案してください。

## PASS 3：具体性と書き手

- 固有名詞、数字、日時、場面、発言の不足
- 書き手が実際に引っ掛かった点
- 迷い、ためらい、不採用案
- 音読したときの語彙と呼吸
- 最終判断を誰が引き受けるか

本人が提供していない感情、経験、会話、五感を創作してはいけません。
不足している場合は、書き足さず、本人へ確認すべき質問を示してください。

弱い表現を機械的に強い断定へ変えないでください。
不確実性、条件、推量が事実なら残してください。

# 出力

1. 総評
2. PASS 1の問題と修正
3. PASS 2の段落診断と再構成案
4. PASS 3で不足している原材料
5. 本人への追加質問
6. 残すべき表現
7. 削るべき表現
8. 修正文
9. 採用した修正と、採用しなかった修正の理由

文章：
（ここに貼る）
```
