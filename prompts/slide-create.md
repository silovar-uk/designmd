# 最重要方針

完成版を一度に作らないでください。

目的、原材料、参照戦略、構成、台本、図の文法、デザインシステム、代表ページ、全体制作を別々の段階として扱ってください。

対話可能な場合は、逆質問、構成案の選択、代表ページの確認で一度止まり、ユーザーの回答または承認を得てから次へ進んでください。

ユーザーが「一括で」「質問不要」「仮定して進めて」と明示した場合のみ、一度の出力で進めて構いません。ただし、その場合も内部では全ステージを順番に通し、置いた前提、比較した案、採用理由、捨てた案を提出時に示してください。

# GATE 0：逆質問

不足情報がある場合、制作へ入る前にユーザーへ逆質問してください。

逆質問は、通常の箇条書きで散らして返さず、ユーザーがそのまま回答を書き込める `text` コードブロックで提示してください。

質問は、必須、推奨、任意に分類し、各質問に `回答：` を付けてください。

既に分かっていることは質問せず、前提として記入してください。

```text
【こちらで置いた前提】
- テーマ：
- 成果物：
- 既に分かっている要望：

【必須】
1. 主な読み手：
   回答：

2. 使用場面：
   選択肢：発表用／閲覧用／会議用／営業用／学習用／Web公開用
   回答：

3. 読後に求める理解・判断・行動：
   回答：

【推奨】
4. 今回いちばん出したい温度：
   選択肢：創業者の原体験／プロダクトローンチ／実験発表／チームのキックオフ／未来像
   回答：

5. ユーモアの強さ：
   選択肢：静かな微笑み／軽いツッコミ／自虐を含む笑い／大胆な実験
   回答：

6. この題材にしかない原材料：
   例：発言、数字、日時、写真、手書き、失敗、固有名詞
   回答：

【任意】
7. 参考にしたい媒体や雰囲気：
   回答：

8. 絶対に避けたい表現：
   回答：
```

対話モードでは、ここで一度止まってください。

# GATE 1：原材料と証拠

ユーザーの回答後、元記事、文字起こし、メモ、データ、会話ログ、研究、画像などから、先に次を抜き出してください。

- 実際の発言
- 固有名詞
- 日時、人数、数値
- 場所や場面
- 失敗、違和感、迷い
- 反対意見
- 根拠となる出典
- 断定できない点
- 制作者が特に引っ掛かった点

次へ分類してください。

- 事実
- 解釈
- 仮説
- 未確認
- 提案
- 演出候補

# GATE 2：参照戦略

参照資料を、雰囲気ではなく次の四役へ分けてください。

1. 骨格：出版物、グリッド、組版、階層
2. 逸脱：実験的タイポグラフィ、ポスター、コラージュ
3. 時間：漫画、映画タイトル、映像、舞台、展示動線
4. 固有素材：元記事、会話、写真、表、手書き、現場記録

各参照について、借りる操作、借りない外見、内容との接続、リスクを示してください。

# GATE 3：構成仮説

いきなり1案に決めず、最低3案を出してください。

各案について、強み、弱み、向いている使用場面、面白味が生まれる場所、誤解のリスク、不採用理由を示してください。

対話モードでは、ここで一度止まり、ユーザーに構成案を選んでもらってください。

# GATE 4：ストーリーボードと台本

採用された構成について、各ページへ役割を割り当ててください。

各ページについて、次を示してください。

- ページの役割
- 読者向け見出し
- 使う原材料
- 読み手の理解がどう変わるか
- 情報密度。低、標準、高
- 速度。静、標準、動
- 前後ページとの関係
- 採用する形式。大きな一文、箇条書き、表、引用、実物、比較図、因果図、時系列など
- 図で表す関係を一文にした「図の一文」
- 読み上げ台本の初稿
- 本編へ表示する情報
- ノート欄へ入れる情報
- 付録へ送る情報
- 制作ログへ残す情報

読み上げ台本は、箇条書きの断片ではなく、実際にそのまま話せる自然な文章で書いてください。

# GATE 5：デザインシステムと図の文法

全ページを作る前に、次を設計してください。

- グリッド、ベースライン、安全余白
- 書体と文字階層
- 出典、注釈、ページ番号の位置
- 色の役割
- 低密度、標準密度、高密度の使い分け
- 資料全体の速度
- 繰り返すモチーフ
- 一つの逸脱ルール
- 図形の意味
- 矢印の動詞
- 囲み、帯、レーンの意味

図形へ変えることで新しい関係が増えない場合、箇条書きや文章を優先してください。

並列、比較、グルーピングには原則として矢印を使わないでください。

# GATE 6：代表ページ

全体を量産する前に、3〜6ページを試作してください。

最低限、次を含めてください。

- 静かなページ
- 標準ページ
- 高密度ページ
- 動きのあるページ
- プレーンな箇条書きページ
- 図形と矢印を使うページ

代表ページでは、次を確認してください。

- 30秒テスト
- 再話テスト
- 図単独テスト
- 本文単独テスト
- 台本の音読テスト
- 実寸閲覧テスト

対話モードでは、ここで一度止まり、ユーザーの確認を得るまで全ページを制作しないでください。

# GATE 7：全体制作

代表ページで承認されたルールを使って全体を制作してください。

ここで新しいスタイルを次々に追加しないでください。必要な例外には、内容上の理由を持たせてください。

PowerPointを制作する場合は、各スライドのノート欄へ、文章形式の読み上げ台本を入れてください。

PDFだけを制作する場合は、ページ番号と対応した台本MD、DOCX、ノート付きPDF、または台本付録を作成してください。

台本のないスライドを完成扱いにしないでください。

# GATE 8：最終レビュー

完成後、以下を確認してください。

- 逆質問を飛ばして早すぎる収束をしていないか
- 比較した構成案と不採用理由があるか
- 代表ページで方向性を検証したか
- 事実、解釈、仮説、提案を区別しているか
- 画像や図が情報上の仕事をしているか
- 図形へ変換して新しい関係が増えたか
- 箇条書きのまま残すべきページを無理にカード化していないか
- 各スライドに文章形式の台本があるか
- 台本がスライド本文の読み上げになっていないか
- 重要な視覚情報を台本でも説明しているか

# 提出時に添えること

- 置いた前提
- 回答された逆質問
- 段階ごとに確定した判断
- 比較した構成案
- 採用した構成の理由
- 捨てた構成案と不採用理由
- 参照した資料と借りた操作
- 借りなかった外見
- 箇条書きのまま残したページと理由
- 図形へ変換したページと理由
- グリッドと文字階層
- 図形と矢印の文法
- 逸脱ルール
- 代表ページで確認したこと
- 各ページの文章形式の台本
- まだ確認が必要な点
