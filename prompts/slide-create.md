# スライド新規作成プロンプト

```text
以下のガイドを必須要件として参照してください。

共通原則：
https://raw.githubusercontent.com/silovar-uk/designmd/main/DESIGN.md

スライド設計：
https://raw.githubusercontent.com/silovar-uk/designmd/main/docs/slides.md

画像・図解：
https://raw.githubusercontent.com/silovar-uk/designmd/main/docs/visuals.md

依頼内容：
（ここに記載）

最初に、次を整理してください。
- 読み手
- 使用場面（発表用／閲覧用／会議用／営業用）
- 読み手にしてほしい判断・理解・行動
- 原材料のうち、事実／解釈／未確定／提案
- この資料でしか伝えられない具体情報

その後、各ページへ「観察・問い・事実・比較・理論・証拠・解釈・反証・判断・提案・次アクション」の役割を割り当ててください。

要件：
- 同一レイアウトを3ページ以上続けない
- カードを使う前に、要素が本当に並列か確認する
- すべてのページをキャッチコピーや強い断言にしない
- 抽象語には、数字、発言、日時、場面のいずれかを近接させる
- 事実、解釈、仮説、提案を見た目でも区別する
- 強い主張の近くに根拠、条件、限界を置く
- 画像や図は、削除すると失われる情報を説明できる場合だけ使う
- 装飾目的の英語ラベル、アイコン、粒子、発光表現を使わない

初稿後、次を自己点検して修正してください。
https://raw.githubusercontent.com/silovar-uk/designmd/main/docs/review-checklist.md

提出時に、採用した構成の理由と、捨てた構成案を簡潔に示してください。
```
