# 最重要方針

完成版を一度に作らないでください。

目的、原材料、参照戦略、構成、台本、図の文法、デザインシステム、代表ページ、全体制作を別々の段階として扱ってください。

対話可能な場合は、逆質問、構成案の選択、代表ページの確認で一度止まり、ユーザーの回答または承認を得てから次へ進んでください。

ユーザーが「一括で」「質問不要」「仮定して進めて」と明示した場合のみ、一度の出力で進めて構いません。ただし、その場合も内部では全ステージを順番に通し、置いた前提、比較した案、採用理由、捨てた案を提出時に示してください。

# GATE 0：逆質問

不足情報がある場合、制作へ入る前にユーザーへ逆質問してください。

逆質問は、通常の箇条書きで散らして返さず、ユーザーがそのまま回答を書き込める `text` コードブロックで提示してください。

空欄だけを並べて、ユーザーへゼロから考えさせないでください。

先に次を整理します。

- すでに確定している前提
- 明示された要望、好み、避けたい表現
- 過去のフィードバック
- 今回の成果物から自然に推測できる条件
- まだ確定していないが、有力な仮説

方向性の質問では、今回の題材に合わせた具体案を2〜4案提示してください。選択肢だけを並べず、推奨案と推奨理由を示してください。

回答欄は `推奨で進める／修正：` を基本にし、ユーザーが違うところだけ直せる形にしてください。

`その他` や案の組み合わせを許可してください。日時、人数、予算、権利条件など、誤ると危険な事実は無理に推測しないでください。

出力例：

# GATE 1：原材料と証拠

ユーザーの回答後、元記事、文字起こし、メモ、データ、会話ログ、研究、画像などから、実際の発言、固有名詞、日時、人数、数値、場面、失敗、違和感、反対意見、出典、断定できない点を抜き出してください。

それらを、事実、解釈、仮説、未確認、提案、演出候補へ分類してください。

# GATE 2：参照戦略

参照資料を、雰囲気ではなく次の四役へ分けてください。

1. 骨格：出版物、グリッド、組版、階層
2. 逸脱：実験的タイポグラフィ、ポスター、コラージュ
3. 時間：漫画、映画タイトル、映像、舞台、展示動線
4. 固有素材：元記事、会話、写真、表、手書き、現場記録

各参照について、借りる操作、借りない外見、内容との接続、リスクを示してください。

# GATE 3：構成仮説

いきなり1案に決めず、最低3案を出してください。

各案について、強み、弱み、向いている使用場面、面白味が生まれる場所、誤解のリスク、不採用理由を示してください。

対話モードでは、ここで一度止まり、ユーザーに構成案を選んでもらってください。

# GATE 4：ストーリーボードと台本

採用された構成について、各ページへ次を割り当ててください。

- ページの役割
- 読者向け見出し
- 使う原材料
- 読み手の理解がどう変わるか
- 情報密度。低、標準、高
- 速度。静、標準、動
- 前後ページとの関係
- 採用する形式。大きな一文、箇条書き、表、引用、実物、比較図、因果図、時系列など
- 図で表す関係を一文にした「図の一文」
- 読み上げ台本の初稿
- 本編へ表示する情報
- ノート欄へ入れる情報
- 付録へ送る情報
- 制作ログへ残す情報

読み上げ台本は、箇条書きの断片ではなく、実際にそのまま話せる自然な文章で書いてください。

# GATE 5：デザインシステムと図の文法

全ページを作る前に、グリッド、ベースライン、安全余白、書体と文字階層、出典、注釈、ページ番号、色の役割、密度、速度、モチーフ、逸脱ルール、図形と矢印の意味を設計してください。

図形へ変えることで新しい関係が増えない場合、箇条書きや文章を優先してください。

並列、比較、グルーピングには原則として矢印を使わないでください。

# GATE 6：代表ページ

全体を量産する前に、3〜6ページを試作してください。

最低限、静かなページ、標準ページ、高密度ページ、動きのあるページ、プレーンな箇条書きページ、図形と矢印を使うページを含めてください。

代表ページでは、30秒テスト、再話テスト、図単独テスト、本文単独テスト、台本の音読テスト、実寸閲覧テストを行ってください。

対話モードでは、ここで一度止まり、ユーザーの確認を得るまで全ページを制作しないでください。

# GATE 7：全体制作

代表ページで承認されたルールを使って全体を制作してください。

ここで新しいスタイルを次々に追加しないでください。必要な例外には、内容上の理由を持たせてください。

PowerPointを制作する場合は、各スライドのノート欄へ、文章形式の読み上げ台本を入れてください。

PDFだけを制作する場合は、ページ番号と対応した台本MD、DOCX、ノート付きPDF、または台本付録を作成してください。

台本のないスライドを完成扱いにしないでください。

# GATE 8：最終レビュー

完成後、以下を確認してください。

- 逆質問を飛ばして早すぎる収束をしていないか
- 逆質問で空欄だけを並べていないか
- 今回の題材に合わせた具体案、推奨案、推奨理由を示したか
- ユーザーが違うところだけ修正できる形だったか
- 比較した構成案と不採用理由があるか
- 代表ページで方向性を検証したか
- 事実、解釈、仮説、提案を区別しているか
- 画像や図が情報上の仕事をしているか
- 図形へ変換して新しい関係が増えたか
- 箇条書きのまま残すべきページを無理にカード化していないか
- 各スライドに文章形式の台本があるか
- 台本がスライド本文の読み上げになっていないか
- 重要な視覚情報を台本でも説明しているか

# 提出時に添えること

- 置いた前提
- 逆質問で提示した案、推奨案、推奨理由
- ユーザーが修正した点
- 段階ごとに確定した判断
- 比較した構成案
- 採用した構成の理由
- 捨てた構成案と不採用理由
- 参照した資料と借りた操作
- 借りなかった外見
- 箇条書きのまま残したページと理由
- 図形へ変換したページと理由
- グリッドと文字階層
- 図形と矢印の文法
- 逸脱ルール
- 代表ページで確認したこと
- 各ページの文章形式の台本
- まだ確認が必要な点
