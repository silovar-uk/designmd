# 最重要方針

完成版を一度に作らないでください。

目的、原材料、参照戦略、構成、台本、図の文法、デザインシステム、代表ページ、全体制作を別々の段階として扱ってください。

対話可能な場合は、指定したステージゲートで一度止まり、ユーザーの回答または承認を得てから次へ進んでください。

ユーザーが「一括で」「質問不要」「仮定して進めて」と明示した場合のみ、一度の出力で進めて構いません。ただし、その場合も内部では全ステージを順番に通し、置いた前提、比較した案、採用理由、捨てた案を提出時に示してください。

# GATE 0：逆質問

まず、成果物の精度に影響する不足情報を確認してください。

不足がある場合は、制作へ入る前にユーザーへ逆質問してください。

質問は最大8問に絞り、次の3分類で示してください。

- 必須：これがないと方向性がずれる
- 推奨：あると品質が上がる
- 任意：こだわりがあれば反映する

確認候補：

1. 読み手は誰ですか。
2. どの場面で、どのように使いますか。発表用、閲覧用、会議用、営業用、共有資料のどれですか。
3. 読み手に、何を判断、理解、実行してほしいですか。
4. 絶対に読みやすく保つべき情報、固定したい要素は何ですか。
5. あえて違和感、読みづらさ、強い表現を許せる場所はどこですか。
6. ユーザーが「ありきたり」「AIっぽい」と感じる表現は何ですか。
7. この題材にしかない原材料は何ですか。発言、数字、日時、写真、手書き、失敗、固有名詞などを確認してください。
8. 資料全体の速度は、徐々に加速、静と動を交互、中盤で爆発、最後へ収束、のどれに近いですか。

質問が不要なほど情報がそろっている場合は、質問を作るために質問しないでください。代わりに、読み取った前提を示してください。

## GATE 0の出力

- 逆質問、または置いた前提
- 読み手
- 使用場面
- 求める判断、理解、行動
- 制約
- 固定するもの
- 変化させてよいもの
- 避ける表現

対話モードでは、ここで一度止まってください。

# GATE 1：原材料と証拠

ユーザーの回答後、元記事、文字起こし、メモ、データ、会話ログ、研究、画像などから、先に次を抜き出してください。

- 実際の発言
- 固有名詞
- 日時、人数、数値
- 場所や場面
- 失敗、違和感、迷い
- 反対意見
- 根拠となる出典
- 断定できない点
- 制作者が特に引っ掛かった点

次へ分類してください。

- 事実
- 解釈
- 仮説
- 未確認
- 提案
- 演出候補

抽象化する前に、スライドへ残すべき原材料を選んでください。

## GATE 1の出力

- 原材料リスト
- 証拠マップ
- この資料でしか使えない具体情報
- 追加確認事項

# GATE 2：参照戦略

参照資料を、雰囲気ではなく次の四役へ分けてください。

1. 骨格：出版物、グリッド、組版、階層
2. 逸脱：実験的タイポグラフィ、ポスター、コラージュ
3. 時間：漫画、映画タイトル、映像、舞台、展示動線
4. 固有素材：元記事、会話、写真、表、手書き、現場記録

各参照について、次を示してください。

- 借りる操作
- 借りない外見
- 今回の内容に適する理由
- 過剰に使った場合のリスク

参考資料が与えられていない場合は、題材と使用場面に基づき、参照すべき媒体や作品の種類を提案してください。有名デザイナーの外見を丸ごと模倣しないでください。

## GATE 2の出力

| 参照 | 役割 | 借りる操作 | 借りない外見 | 内容との接続 | リスク |
|---|---|---|---|---|---|

# GATE 3：構成仮説

いきなり1案に決めず、最低3案を出してください。

例：

- A案：具体的な場面から始める
- B案：データや研究結果から始める
- C案：誤解、反論、違和感から始める
- D案：同じ素材を二つの見方で読み直す

各案について、次を示してください。

- 強み
- 弱み
- 向いている使用場面
- 面白味が生まれる場所
- 誤解のリスク
- 採用しない場合の理由

最もきれいな案ではなく、目的、読み手、原材料に最も適した案を推奨してください。

## GATE 3の出力

- 構成案3案以上
- 推奨案
- 推奨理由
- 捨てる案と不採用理由

対話モードでは、ここで一度止まり、ユーザーに構成案を選んでもらってください。

# GATE 4：ストーリーボードと台本

採用された構成について、各ページへ役割を割り当ててください。

- 観察
- 問い
- 事実
- 比較
- 理論
- 証拠
- 解釈
- 反証
- 判断
- 提案
- 次アクション
- 間
- 転換
- 回収

各ページについて、次を示してください。

- ページの役割
- 使う原材料
- 読み手の理解がどう変わるか
- 情報密度。低、標準、高
- 速度。静、標準、動
- 前後ページとの関係
- 図で表す関係を一文にした「図の一文」
- 読み上げ台本の初稿
- 図の読み方
- 前ページからの接続文
- 次ページへの接続文

読み上げ台本は、箇条書きの断片ではなく、実際にそのまま話せる自然な文章で書いてください。

台本には、ページの結論または問い、図の読み順、主要な図形の役割、矢印が表す関係、条件、限界、次ページへの橋渡しを含めてください。

この段階では、完成レイアウトや細かな装飾を作らないでください。

「一枚一メッセージ」を「一枚一断言」と解釈しないでください。問い、比較、未確定事項、反証も一つのメッセージです。

# GATE 5：デザインシステムと図の文法

全ページを作る前に、次を設計してください。

## 秩序

- グリッド。6列、12列など、採用理由も示す
- ベースラインまたは垂直方向の基準
- 書体と文字階層
- 安全余白
- 出典、注釈、ページ番号の位置
- 色の役割

## 変化

- 低密度、標準密度、高密度の使い分け
- 資料全体の速度
- 繰り返すモチーフ
- 一つの逸脱ルール

## 図の文法

- 図形は何を表すか
- 矢印はどの動詞を表すか
- 囲み、帯、レーンはどの文脈を表すか
- 色、太さ、大きさは何を修飾するか
- 主方向は左から右、上から下のどちらか
- 同じ形、同じ線種をどの意味へ統一するか

矢印を置く前に、各接続を「AがBへ何をする」と一文で説明してください。

次を守ってください。

- 並列、比較、グルーピングには原則として矢印を使わない
- 階層や分解には、必要がなければ矢印頭を付けない
- 時間、因果、受け渡し、依存関係を同じ矢印で混同しない
- 一つの図では主方向を統一する
- PowerPointでは図形へ接続するコネクタを使う
- 線の交差と複数回の折れ曲がりを避ける
- 分岐には条件を書く
- 循環矢印は本当に戻る仕組みがある場合だけ使う
- 太いブロック矢印は、大きな移行または変換に限定する
- 矢印を削除しても意味が変わらない場合は削除する

逸脱ルール例：

- 予想が外れるページだけ要素がグリッドから外れる
- 反証ページだけ左右を反転する
- 本音の発言だけ欄外へ出す
- 転換点だけ一語が画面外へ切れる
- 同じ図を意味を変えて再登場させる

逸脱は、変化をつけるためではなく、内容上の転換を可視化するために使ってください。

# GATE 6：代表ページ

全体を量産する前に、3〜5ページを試作してください。

最低限、次を含めてください。

1. 静かなページ：問い、引用、章扉など
2. 標準ページ：主張と根拠、比較など
3. 高密度ページ：研究、表、反証など
4. 動きのあるページ：転換、予想外し、結論など
5. 図形と矢印を使うページ：プロセス、因果、引き継ぎなど

単独ページだけでなく、連続したときの速度と変化を確認してください。

代表ページでは、台本を実際に読み上げながら確認してください。

確認項目：

- 同じ世界に属して見えるか
- 役割に応じて形が変わっているか
- グリッドがテンプレート化していないか
- 逸脱が内容と結びついているか
- 題材固有の素材が主役になっているか
- 静、密、動の差があるか
- 画像や図を削除すると失われる情報を説明できるか
- 図形を名詞、矢印を動詞として説明できるか
- 視線の順序と台本の順序が一致するか
- 台本で触れない図形や矢印がないか
- スライド本文をそのまま読み上げていないか

## GATE 6の出力

- 代表ページ3〜5枚
- 各ページの文章形式の台本
- 採用する操作
- 修正する操作
- 捨てる操作
- 全体へ展開する際の注意

対話モードでは、ここで一度止まり、ユーザーの確認を得るまで全ページを制作しないでください。

# GATE 7：全体制作

代表ページで承認されたルールを使って全体を制作してください。

ここで新しいスタイルを次々に追加しないでください。必要な例外には、内容上の理由を持たせてください。

内容の種類に応じてページ構造を変えてください。

- 引用：原文、発言者、日時、出所を近くに置く
- 比較：比較軸をそろえ、差分だけを強調する
- 因果：原因、中間過程、結果を示し、推測と確認済みを分ける
- 時系列：方向を統一し、日時と転換点を示す
- 研究：対象、方法、結果、限界を同じページまたは隣接ページで示す
- 提案：現状、問題、対応、効果、リスク、判断事項を示す
- プロセス：同じ粒度の行動を一方向へ並べる
- 引き継ぎ：スイムレーンで担当を分け、矢印へ渡すものを示す
- 階層・分解：枝線を使い、因果と誤解される矢印頭を付けない

同一レイアウトを3ページ以上続けないでください。
カードを使う前に、要素が本当に並列か確認してください。

PowerPointを制作する場合は、各スライドのノート欄へ、文章形式の読み上げ台本を入れてください。

PDFだけを制作する場合は、ページ番号と対応した台本MD、DOCX、ノート付きPDF、または台本付録を作成してください。

台本のないスライドを完成扱いにしないでください。

# 画像・図解の使用条件

画像や図は、削除すると失われる情報を説明できる場合だけ使ってください。

避けるもの：

- 装飾目的の英語ラベル
- 意味のないアイコン
- 発光する人物
- 粒子
- ワイヤーフレーム
- ニューラルネットワーク風の抽象表現
- 抽象概念を抽象画像へ置き換えただけのビジュアル
- 紙テクスチャ、版ずれ、手書き風など、人間らしさを演出するだけの加工
- 意味を定義できない矢印
- 並列要素を無理につなぐ矢印
- 装飾のための太いブロック矢印

優先するもの：

- 原文引用
- 実データ
- 実際の場面
- 比較図
- 因果図
- 時系列
- 研究の対象、方法、結果、限界
- 題材固有の写真、表、手書き、記号
- 意味を動詞で説明できる接続

# GATE 8：最終レビュー

完成後、以下のレビュー基準に沿って自己点検し、重大な問題を修正してください。

https://raw.githubusercontent.com/silovar-uk/designmd/main/docs/review-checklist.md

特に確認してください。

- 原材料から完成版へ早すぎる収束をしていないか
- 比較した構成案と不採用理由があるか
- 代表ページで方向性を検証したか
- 抽象語が具体語より多くないか
- すべてのページがキャッチコピーになっていないか
- 事実、解釈、仮説、提案を区別しているか
- 強い主張の近くに根拠、条件、限界があるか
- 画像や図が情報上の仕事をしているか
- 参照先の外見を表面的に模倣していないか
- グリッドが同じ箱の反復になっていないか
- 逸脱に内容上の理由があるか
- 静、標準、密、動の速度差があるか
- 読み手に固有の情報が入っているか
- なぜこの構成なのか説明できるか
- 各矢印を動詞で説明できるか
- 同じ形と線種が同じ意味になっているか
- 比較、並列、所属へ不要な矢印を使っていないか
- 主方向が統一されているか
- 線の交差、過剰な折れ曲がり、浮いた矢印がないか
- 各スライドに文章形式の台本があるか
- 台本がスライド本文の読み上げになっていないか
- 台本の説明順と図の視線順が一致しているか
- 重要な視覚情報を台本でも説明しているか

# 提出時に添えること

成果物と一緒に、次を簡潔に示してください。

- 段階ごとに確定した判断
- 比較した構成案
- 採用した構成の理由
- 捨てた構成案と不採用理由
- 参照した資料と借りた操作
- 借りなかった外見
- グリッドと文字階層
- 図形と矢印の文法
- 密度モード
- 逸脱ルール
- 代表ページで確認したこと
- 各ページの文章形式の台本
- ガイドに照らして修正した点
- まだ確認が必要な点
