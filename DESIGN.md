# AIくささを減らす制作原則

> designmd `2026.07.18-beta`

この文書は、AIへ一つだけ参照させるための自己完結ダイジェストである。詳細な正典は`docs/`配下にある。

目標は、AI使用の痕跡を隠すことではない。

> 誰が、何を見て、何を重要と判断し、何を捨てたのかが伝わる成果物をつくる。

内容に対して、最小で正しい形式を選ぶ。立派さ、項目数、工程数を完成条件にしない。

---

## 1. 判断の原則

- スタイルを先に決め、内容を流し込まない
- 固有の発言、数字、失敗、違和感を一般論へ丸めない
- 事実、解釈、仮説、提案、演出を区別する
- 強い主張の近くに、根拠、条件、限界を置く
- AIと人間が問いを共同生成し、人間が採用する問いと説明責任を持つ

**AIも問いを問い直す。**

与えられた問いを自動的に正解として扱わない。AIは、前提、根拠、本当に変えたい状態、外部文脈、代替問いを確認する。

原則として次の三案を比較する。

1. ブリーフ忠実案
2. 欲望へ持ち上げた再定義案
3. 前提を採用しない反転・別軸案

AIは問いを生成し、比較し、反論できる。ただし、刺激的であることを採用理由にしない。どの問いへ時間、予算、信用を賭けるかは人間が決める。

緊急告知、安全情報、法令対応、決定済み事項の正確な伝達では、問いを広げず、条件と誤解防止を優先する。

詳細：[`docs/question-audit.md`](docs/question-audit.md)

**共通基盤と案件固有を分ける。**

designmdが決めるのは、判断の順序、確認方法、最低品質である。案件ごとの書体、色、余白、語り口、逸脱ルールは、案件別スタイルガイドで決める。

```text
案件名：
読み手／使用場面：
資料モード：説明・報告／説得・提案／読み物・体験／混合
読後の変化：
採用した問い：
中心メッセージ：
固有素材：
語り口：
タイトル設計：内容説明型／解釈併記型／提案・判断型の使い分け
強いタイトルを使うページ：
視覚の骨格：
一つの逸脱ルール：
アクセシビリティ最低条件：
固定するもの／変えてよいもの：
避ける表現：
最終判断者：
```

**一度に見せる認知単位は原則5つ以下。**

分類、段階、カード、タブ、選択肢を同じ階層へ6つ以上並べる場合は、必要性を説明する。情報を削るのではなく、最初の入口を3〜5群へ束ね、詳細は下位のチェック、付録、制作ログへ送る。

- 見出しと同じ意味の副題は削る
- 日本語見出しと同じ意味の英語ラベルを重ねない
- 制作上のタグを読者向け見出しにしない
- 数をそろえるために内容を分割しない
- 副題を削って意味が変わらなければ削る

## 2. 五段階でつくる

### 1. 定める

読み手、使用場面、読後の変化、提出形式、閲覧環境、固定条件を決める。

制作へ入る前に、AIと人間で問いを監査する。

1. 前提：何を事実として扱っているか
2. 根拠：事実、仮説、慣習、未確認のどれか
3. 欲望：解決後に本当は何を変えたいか
4. 文脈：生活者、競合、組織、社会、時代を加えると何が変わるか
5. 代替問い：元の問いを採用しない場合、何を問うか

逆質問では空欄を並べず、会話から読み取った前提、問いの候補、具体案、推奨案、理由を先に示す。

### 2. 集める

発言、数字、日時、場所、失敗、反対意見、画像、表、出典、未確認事項を集める。

ブリーフ内だけでなく、問いを支える生活者、競合、組織、制度、時代の材料と、反証材料も集める。

事実、解釈、仮説、未確認、提案、演出候補を同じ確信度で扱わない。

### 3. 組み立てる

採用した問いに答える中心メッセージを短い一文で決め、構成案を最低三案比較する。

ページ・節の上位役割は五つに束ねる。

1. 見る：観察、事実、証拠
2. 比べる：問い、比較、反証
3. 考える：理論、解釈
4. 決める：判断、提案
5. 進める：次アクション、間、転換、回収

詳細タグは制作ログへ置き、読者向けの副題にはしない。

情報は本編、台本、付録・根拠、制作ログの四層へ分ける。

スライドでは、資料モードとページ役割を掛け合わせてタイトルを決める。提案資料であっても、事実ページは内容説明型、分析ページは解釈併記型、方針ページは提案・判断型を基本とする。すべてのページをキャッチコピーや強い断言へ変換しない。

タイトルはレイアウト前に仮置きし、全ページのタイトルだけを一覧して、事実から解釈、提案、次アクションへ論理が進むか確認する。

詳細：[`docs/slide-title-strategy.md`](docs/slide-title-strategy.md)

### 4. つくる

参照先を骨格、逸脱、時間、固有素材へ分け、借りる操作と借りない外見を示す。

全体制作前に3〜5ページを試す。静、標準、密、動が同じ世界に属し、役割に応じて形が変わるか確認する。

対話できる場合は、問いの監査、構成案、代表ページで一度止まる。

### 5. 確かめる

AIらしさ、アクセシビリティ、旧版の機能保存、実寸、音読、出典、条件、限界を確認する。

問いについても確認する。

- 採用した問いは証拠と結びついているか
- 元の問いを否定すること自体が目的になっていないか
- 刺激的だからという理由だけで選んでいないか
- 採用しなかった問いと理由が残っているか
- 最終判断者が問いへの責任を引き受けているか

AIくささは次の五群で見る。

- 言葉と材料が平均化する
- 形式が内容より先に立つ
- 構造が均一・過密になる
- 作り方が前へ出る
- 人の判断と機能が消える

## 3. 文章

日本語文章は四回編集する。

**PASS 1：表面**

記号、Markdown、半角スペース、定型句、空句、同じ文末、内容を増やさない予告・総括を取る。

**PASS 2：論証**

一段落一トピック、主張と根拠、因果、条件、限界、段落の受け渡しを確認する。必要なら削除、統合、移動する。

**PASS 3：認知リズム**

文章の進行だけを実況する文を削り、対象、数字、発言、因果、判断へ戻す。問い、仮の見立て、反論、後で説明する約束は回収位置を管理する。

認知モードは観察、推論、保留、反証、固定、適用を使うが、固定配列にしない。材料がない場所へ迷い、告白、反証を創作しない。

長い読み物では、事象と心象を往復する。

- 事象：出来事、作品、発言、数字、歴史など、書き手の外にある材料
- 心象：材料に触れた書き手の実際の反応、違和感、判断変更、保留

```text
事象 → 心象 → 別の事象 → 見立ての更新
```

別作品、記憶、周辺知識へ脱線する場合は、戻ったときに元の対象の見え方が変わることを条件にする。笑いは注意を回復するために使い、直後に対象、事実、問いのいずれかへ戻る。冒頭の言葉や比喩を結びで繰り返すだけでなく、後半の材料によって意味を更新して回収する。

固有の文体、駄洒落、自虐、架空会話を模倣しない。書き手が持っていない反応、失敗、会話を創作しない。

**PASS 4：身体**

実際の場面、発言、数字、本人の語彙と呼吸へ戻す。書き手が提供していない感情、経験、会話、五感を創作しない。

文書種別によって適用強度を変える。

- 全工程：コラム、解説、インタビュー、企画文、学習資料
- 一部：提案書、社内説明、調査レポート、プレゼンノート、プレスリリース
- 対象更新のみ：緊急告知、安全情報、法務、手順書、議事録、短いメール

結論、重要条件、注意事項を、読ませる演出のために遅らせない。

## 4. 視覚と形式

**画像の優先順位**

1. 実際の現場写真・画面
2. 原文・発言・資料の抜粋
3. 実データの図表
4. 内容固有の説明図
5. 権利確認済み素材

生成画像、汎用アイコン、装飾は、上の素材で目的を果たせない場合だけ使う。削除して失われる情報が「見栄え」だけなら削除する。

**形式は仕事で選ぶ。**

- 箇条書き：並列の観察、条件、注意
- カード：独立した入口で、画像・説明・操作が一つの主題になる
- 表：同じ軸で正確な値・条件・担当を比較する
- 画像・実物：対象、場面、証拠、空間を示す
- 図：因果、流れ、分岐、位置、変化を新たに示す

見出しは長文の階層として使い、すべての節へ副題を付けない。

**スライドタイトルは、資料目的とページ役割で決める。**

- 説明・報告資料：対象と内容を正確に示す
- 説得・提案資料：事実ページは説明し、提案ページは結論や推奨方針を示す
- 読み物・体験資料：問い、比喩、キャッチコピーを使用できる

タイトルの強度は、内容説明型、解釈併記型、提案・判断型の三段階で選ぶ。役割を果たせる範囲で最も弱い強度を使い、強いタイトルは中心提案、章扉、転換、判断ページへ集中させる。

タイトル、キーメッセージ、本文を分ける。

- タイトル：何について、何を示すページか
- キーメッセージ：掲載情報から何を読み取るか
- 本文・図表：事実、根拠、条件、例外
- ノート：背景、因果、留保、前後ページとの接続

「AではなくB」、命令形、内容を特定できない比喩、短い断言を複数ページで反復しない。タイトルだけを一覧し、各ページの対象と、資料全体の論理を追えるか確認する。

詳細：[`docs/slide-title-strategy.md`](docs/slide-title-strategy.md)

**図の文法**

- 図形＝名詞
- 矢印＝動詞
- 囲み、帯、レーン＝文脈
- 色、太さ、大きさ＝修飾語

矢印は「AがBへ何をする」と言える場合だけ使う。

**学習資料**

七つの詳細機能を、入口では五つへ束ねる。

1. 知る：定義、識別
2. 分かる：理由、仕組み
3. やる：手順
4. 変える：状況変化、失敗条件
5. 使いこなす：転用、自力判断

## 5. 品質、納品、責任

**アクセシビリティ**

- 情報の順序を位置や色だけに依存させない
- 見出しと本文の階層を意味に合わせる
- 図、写真、表、動画の要点を文章でも説明する
- 実寸、拡大、読み上げ、キーボード、別端末で確認する
- designmdを規格適合の保証として扱わない

**PowerPoint**

- 全スライドのノート欄へ文章形式の台本を入れる
- 説明文は常体、だ・である調にする
- 本文の読み上げではなく、背景、因果、条件、限界、接続を補う
- PPTXを完成させ、その最終版からPDFを作る
- PDF化後も全ページの文字切れ、重なり、フォント、画像、余白を確認する
- ユーザーが片方だけを指定しない限り、PPTXとPDFをセットで納品する

**最終レビュー**

- 読み手、使用場面、資料モード、読後の変化、採用した問いが明確か
- 採用した問いを支える証拠と反証があるか
- 同じ階層に6つ以上の分類、副題、選択肢がないか
- 副題を削っても意味が変わらない箇所がないか
- 形式が内容より先に立っていないか
- 旧版の理解、判断、行動、再現性を失っていないか
- 最終判断者と未確認事項が明確か
- タイトルの強度が、資料モードとページ役割に一致しているか
- 事実ページが、根拠以上の断言になっていないか
- 提案ページが、単なる内容紹介で終わっていないか
- タイトルだけを一覧して、各ページの内容と資料構造が分かるか
- 強いタイトルが、中心提案、転換、判断ページへ限定されているか
- 長い読み物で、事象と心象の片方だけが続いていないか
- 脱線の後、元の対象の見方を更新して帰還しているか
- 笑いの直後に、対象、事実、問いのどこへ戻るかを指せるか

**制作ログ**

```text
資料モード：
読後の変化：
依頼された問い：
三つの代替問い：

採用しなかった問いと理由：
中心メッセージ：
残すべき原材料：
構成案と採用・不採用理由：
五つの上位役割：
タイトル強度の使い分け：
強いタイトルを使うページ：
タイトル一覧テスト：
本編／台本／付録／制作ログ：
案件別スタイルガイド：
認知単位が6つ以上ある箇所と理由：
削除した副題・ラベル：
事象／心象／脱線の帰還位置：
アクセシビリティ前提：
実寸・音読・操作テスト：
未確認事項／最終判断者：
```
