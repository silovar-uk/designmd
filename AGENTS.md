# Repository instructions for AI agents

このリポジトリで文章、スライド、画像、プロンプトを追加・修正するAIは、最初に [`DESIGN.md`](DESIGN.md) を読むこと。

タスク別の追加参照先：

- 文章：`docs/writing.md`
- 日本語のAIくささ：`docs/japanese-ai-writing-practices.md`
- スライド：`docs/slides.md`
- 図形・矢印・台本：`docs/diagram-shapes.md`
- 画像・図解：`docs/visuals.md`
- 段階制作：`docs/staged-production.md`
- 参照戦略：`docs/reference-strategy.md`

- 最終レビュー：`docs/review-checklist.md`

## 作業時の必須事項

1. 読み手、使用場面、判断事項を明示する。
2. 事実、解釈、仮説、提案を混同しない。
3. 原材料の具体性を残す。
4. 完成版を一度に作らず、目的、原材料、構成、台本、システム、代表ページ、全体制作を分ける。
5. 対話可能な場合は、逆質問、構成案の選択、代表ページの確認で一度止まる。
6. 構成案を最低3案比較し、採用理由と不採用理由を示す。
7. 参照先を、骨格、逸脱、時間、固有素材へ分ける。
8. 参照先から借りる操作と、借りない外見を示す。
9. 同一テンプレートを内容に関係なく反復しない。
10. グリッドを同じ箱の反復ではなく、情報間の関係を作る文法として使う。
11. 静、標準、密、動の代表ページを全体制作前に確認する。
12. 独自性やダイナミックさは、秩序の上に置く一つの逸脱ルールで作る。
13. 図形は名詞、矢印は動詞、囲み・レーンは文脈として定義する。
14. 矢印を置く前に、接続を「AがBへ何をする」と文章で説明する。
15. 並列、比較、グルーピングへ不要な矢印を使わない。
16. 同じ形と線種は、同じ意味へ統一する。
17. 線の交差、過剰な折れ曲がり、浮いた矢印を避ける。
18. PowerPointでは、各スライドのノート欄へ文章形式の読み上げ台本を入れる。
19. 台本には、図の読み順、主要な矢印の関係、条件、限界、前後ページへの接続を含める。
20. 台本で触れない図形や矢印は削除候補とする。
21. 画像・図の情報上の役割を説明する。
22. 採用・棄却理由を示す。
23. 完成後、`docs/review-checklist.md` に沿って自己点検する。

## 対話できない場合

一度の出力を求められた場合も、内部では各ステージを順番に通す。

提出時に次を示す。

- 置いた前提
- 比較した構成案
- 採用理由
- 参照した資料と借りた操作
- グリッドと文字階層
- 図形と矢印の文法
- 密度モード
- 逸脱ルール
- 代表ページで確認したこと
- 各ページの文章形式の台本
- 捨てた案
- 未確認事項
